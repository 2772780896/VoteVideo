# Video 后端全流程

## 数据流总览

```
GET /api/video/main?page=2&sort=-viewCount
  │
  ▼
videoRoutes.js ── 路由映射 + optionalAuth 中间件
  │
  ▼
videoController.js ── 提取 req.query / req.user，调用 service，用 sendList 输出
  │
  ▼
videoService.js ── 业务编排（本例直接委托 baseService）
  │
  ├─► baseService.getListData()
  │     ├─ parsePagination / parseSortParam
  │     ├─ Prisma findMany + count（并行）
  │     ├─ videoTransformer.transformVideoData()（逐条转换）
  │     └─ interactionService.mergeInteractions()（合并交互状态）
  │
  ▼
utils/response.js (sendList) ── { code:200, data:{ items:[...], total:128 } }
```

---

## 共享机制

以下机制在多个 API 中复用，不再逐一展开。

### 可选认证 (optionalAuth)

`middleware/authMiddleware.js`

- 检查 `Authorization: Bearer <token>` 请求头
- 有效 token → 设置 `req.user = { uid, username, iat, exp }`
- 无 token 或无效 token → 静默跳过（`req.user` 保持 `undefined`），不阻断请求
- 视频路由通过 `router.use(optionalAuth)` 统一挂载

### 数据转换 (transformVideoData)

`services/transformers/videoTransformer.js`

纯函数，不查数据库。将 Prisma Video 对象转为前端友好格式：

| 输出字段 | 来源 |
|---------|------|
| vid, coverUrl, title, viewCount, commentCount, duration | 直接映射 |
| likeCount, favouriteCount, reshareCount | 默认 0 |
| date | `formatDate()` → `"YYYY-MM-DD"` |
| isLiked/isDisliked/isFavourited/isReshared | 默认 `false`，后续由 interactionService 覆盖 |
| videoUrl | 仅 `includeVideoUrl: true` 时输出（详情页） |
| uploader | `{ uid, userName, profilePictureUrl, isFollowing: false }` |
| tagList | 仅 `includeTags: true` 时，将 videoTags 中间表拍平为 `[{ tid, tagName, ... }]` |

### 交互状态合并 (mergeInteractions)

`services/interactionService.js`

1. 从 items 中提取 ID 数组
2. 按 `INTERACTION_CONFIG` 配置，并行查询 UserLike / UserFavourite / UserReshare 表
3. 构建 `{ [id]: { isLiked, isFavourited, isReshared, isDisliked } }` 映射
4. `Object.assign` 原地覆盖 transformer 设置的默认 `false` 值

**为什么不能在主查询中 include？** 这些是多态关联表（通过 `type` + `item_id` 定位），无法用 Prisma 外键 join。

### 统一响应 (utils/response.js)

| 函数 | 用途 | 输出格式 |
|------|------|---------|
| `sendSuccess(res, data, msg, status)` | 详情/特殊响应 | `{ code, message, data }` |
| `sendList(res, result)` | 列表响应 | `{ code: 200, data: { items, total } }` |
| `sendError(res, error, msg)` | 错误响应 | 业务错误用 `error.statusCode`，否则 500 |

## API 1：视频列表

```
GET /api/video?page=2&element=16&sort=-viewCount&q=关键词
GET /api/video/main  （同上，首页入口）
```

**路由**：`/` 和 `/main` 都映射到 `videoController.getVideoList`

**Controller**：
- 提取参数：`page`(1), `element`(16), `sort`("-date"), `q`(""), `uid`, `following`
- `following === 'true'` 转布尔值
- `currentUid = req.user?.uid || null`
- 调用 `videoService.getVideoListData(options)` → `sendList(res, result)`

**Service**：直接委托 `baseService.getListData(options)`，不额外加工。

**baseService.getListData 内部**：
1. `parsePagination(2, 16)` → `{ skip: 16, take: 16 }`
2. `parseSortParam('-viewCount')` → `{ viewCount: 'desc' }`
3. 构建 where：`q` → `title contains`，`uid` → `uploader_uid =`，`following` → 先查关注列表再 `uploader_uid in [...]`
4. `Promise.all` 并行执行 `findMany`（含 uploader + videoTags include）和 `count`
5. 逐条调 `transformVideoData(video, { includeTags: true })`
6. 若有 `currentUid`，调 `mergeInteractions` 覆盖交互状态

**返回**：`{ data: [视频对象...], total: 数字 }`

## API 2：视频详情

```
GET /api/video/:vid
```

**路由**：`/:vid` 必须在 `/main` 和 `/related` **之后**定义，避免路径参数误匹配。

**Controller**：
- `vid = parseInt(req.params.vid)`
- `currentUid = req.user?.uid || null`
- 调用 `videoService.getVideoDetailData(vid, currentUid)` → `sendSuccess(res, videoItem)`

**Service** (`getVideoDetailData`)：
1. **播放量 +1**：`prisma.video.update({ viewCount: { increment: 1 } })`
2. **查数据**：`baseService.getItemData(vid, { throwIfNotFound: true })` — 不存在时抛 404
3. **转换**：`transformVideoData(video, { includeVideoUrl: true, includeTags: true })` — 详情页需要视频地址和标签
4. **交互状态**：若有 `currentUid`，调 `mergeInteractions('video', [videoItem], 'vid', currentUid)`

**返回**：单个完整视频对象（含 videoUrl、tagList、交互状态）

## API 3：相关推荐（Tag 匹配）

```
GET /api/video/related?vid=10&page=1&element=5&sort=-date
```

**Controller**：
- 兼容两种参数名：`req.query.vid || req.query.videoId`（前端通用组件用 videoId）
- 调用 `videoService.getRelatedVideosData(options)` → `sendList(res, result)`

**Service** (`getRelatedVideosData`)：
- 委托 `baseService.getRelatedData(options)`，但覆盖 transformFunction 为 `includeTags: false`（推荐卡片不需要标签列表）
- 映射 `options.vid` → `options.currentId`

**baseService.getRelatedData 内部**（视频专属 tag 匹配逻辑）：

```
1. 查当前视频的 tag（通过 videoTags 中间表）
   → 如 vid=10 有 [React, Node.js, JavaScript]

2. 查 VideoTag 表中有相同 tag 的其他视频（排除自己）
   → 统计每个视频的共同 tag 数（countMap）
   → 如 vid=1: 2个共同tag, vid=3: 1个, vid=5: 1个

3. 按共同 tag 数降序排列，取前 element 个 ID
   → [1, 3, 5]

4. 按排名顺序查完整数据（findMany + include）
   → 用 Map 重新排序（findMany 不保证返回顺序）

5. 若当前视频无 tag → 回退为排除自己后按默认排序取最新
```

**返回**：`{ data: [按匹配度排序的视频...], total: 匹配视频总数 }`

## API 4：轮播图

```
GET /api/picture/carousel?number=5
```

最简单的接口，不经过 baseService：

- Controller 提取 `number`（默认 5）
- Service 直接查 Prisma：按 `viewCount` 降序取前 N 条，仅选 `vid, coverUrl, title`
- `transformToCarouselItems` 转换为 `[{ id: vid, src: coverUrl }]`
- `sendSuccess(res, { items: [...] })`

不涉及交互状态。
