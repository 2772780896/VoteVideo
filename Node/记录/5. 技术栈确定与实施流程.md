## 一、技术栈确定

### 1.1 核心决策

| 技术点 | 选择 | 理由 |
|--------|------|------|
| **数据库** | SQLite | 最轻型，文件型，无需服务器，适合学习和小型项目 |
| **ORM** | Prisma | 类型安全，迁移管理方便，你的笔记中有相关记录 |
| **Web 框架** | Express.js | 生态丰富，你的笔记中有详细记录 |
| **认证** | JWT (jsonwebtoken) | 无状态认证，适合前后端分离 |
| **密码加密** | bcrypt | 行业标配，你的笔记中有记录 |
| **文件上传** | multer | Express 生态标准方案 |

**决策：放弃原生 SQL，直接使用 ORM（Prisma）**

## 二、为什么选这个技术栈？

### 2.1 为什么选 SQLite？

**你的需求**：最轻型数据库

**对比**：

| 特性 | SQLite | MySQL | PostgreSQL |
|-------|--------|--------|-------------|
| 安装难度 | ✅ 零安装（文件型） | ❌ 需要安装服务器 | ❌ 需要安装服务器 |
| 配置难度 | ✅ 零配置 | ❌ 需要配置用户/权限 | ❌ 需要配置用户/权限 |
| 文件存储 | ✅ 单个 `.db` 文件 | ❌ 需要数据目录 | ❌ 需要数据目录 |
| SQL 支持 | ✅ 支持大部分标准 SQL | ✅ 完整支持 | ✅ 完整支持 |
| 多对多关系 | ✅ 支持（JOIN + 中间表） | ✅ 支持 | ✅ 支持 |
| ORM 支持 | ✅ Prisma/Sequelize 都支持 | ✅ 支持 | ✅ 支持 |
| 并发性能 | ❌ 差（适合低并发） | ✅ 好 | ✅ 好 |
| 生产环境 | ⚠️ 适合小型项目 | ✅ 适合中大型 | ✅ 适合中大型 |

**结论**：SQLite 完全满足你的需求（学习项目 + 低并发）。

---

### 2.2 为什么选 Prisma 而不是 Sequelize？

**你的笔记情况**：
- ✅ 你的笔记 `16.node-全栈手册.md` 中有 **Prisma** 的记录
- ❌ 你的笔记中**没有 Sequelize** 的记录

**对比**：

| 特性 | Prisma | Sequelize |
|-------|--------|-----------|
| 类型安全 | ✅ 自动生成 TypeScript 类型 | ⚠️ 需要手动定义类型 |
| 模型定义 | ✅ `schema.prisma`（声明式） | ❌ 需要写 JavaScript 类 |
| 迁移管理 | ✅ `prisma migrate`（自动生成） | ⚠️ `sequelize-cli`（配置复杂） |
| 关联查询 | ✅ `include: { tags: true }` | ⚠️ 需要手动处理 |
| 学习曲线 | ✅ 较平缓 | ❌ 较陡峭 |
| 你的笔记 | ✅ 有记录 | ❌ 无记录 |

**结论**：Prisma 更适合你（有笔记 + 类型安全 + 迁移方便）。

---

### 2.3 为什么放弃原生 SQL？

**原计划的混合模式**：
```
简单 CRUD → ORM
复杂查询   → 原生 SQL
```

**放弃原因**：
1. **学习成本高**：需要同时学 ORM 语法和 SQL 语法
2. **代码风格不统一**：有些地方用 ORM，有些地方用 SQL
3. **Prisma 足够强大**：大部分查询（包括联表）都可以用 Prisma 实现
4. **真正复杂的查询很少**：VoteVideo 的查询主要是 CRUD + 简单联表

**什么时候需要原生 SQL？**
- 复杂报表（多表聚合）
- 窗口函数（排名）
- 递归查询（你的评论递归可以用 Prisma 的递归关系实现）

**结论**：放弃原生 SQL，直接用 Prisma。如果真的遇到 Prisma 无法处理的查询，再考虑原生 SQL。

### 3.3 同步数据库

```bash
# 根据 schema.prisma 生成 Prisma Client
npx prisma generate

# 创建迁移文件并应用到数据库
npx prisma db push

# 或者用 migrate（更规范，但学习成本高）
npx prisma migrate dev --name init
```

**`db push` vs `migrate dev`**：
- `db push`：直接同步数据库（适合快速开发）
- `migrate dev`：生成迁移文件（适合团队协作和生产环境）

**推荐**：开发阶段用 `db push`，上线前改用 `migrate`。

### 3.4 Prisma CRUD 核心语法

#### 查询（Read）

```javascript
const prisma = new PrismaClient();

// 查询所有用户
const users = await prisma.user.findMany();

// 查询单个用户（带条件）
const user = await prisma.user.findUnique({
  where: { uid: 1 }
});

// 查询带关联
const video = await prisma.video.findUnique({
  where: { vid: 123 },
  include: {
    uploader: true,    // 关联用户
    videoTags: {
      include: {
        tag: true       // 关联标签
      }
    }
  }
});

// 分页 + 排序
const videos = await prisma.video.findMany({
  orderBy: { createdAt: 'desc' },
  skip: 0,       // OFFSET
  take: 16,      // LIMIT
});

// 条件查询（搜索）
const videos = await prisma.video.findMany({
  where: {
    OR: [
      { title: { contains: '关键词' } },
      { uploader: { username: { contains: '关键词' } } }
    ]
  }
});
```

### 3.6 认证中间件（结合 JWT）

```javascript
// middlewares/auth.js
const jwt = require('jsonwebtoken');
const prisma = new PrismaClient();

const authMiddleware = async (req, res, next) => {
  // 从 Authorization Header 读取 token
  const authHeader = req.headers.authorization;
  if (!authHeader) {
    return res.status(401).json({ code: 401, message: '请先登录' });
  }

  const token = authHeader.split(' ')[1];

  try {
    // 验证 token
    const decoded = jwt.verify(token, process.env.JWT_SECRET);

    // 查询用户是否存在
    const user = await prisma.user.findUnique({
      where: { uid: decoded.uid }
    });

    if (!user) {
      return res.status(401).json({ code: 401, message: '用户不存在' });
    }

    // 附加到 req 对象
    req.user = user;
    next();
  } catch (err) {
    return res.status(401).json({ code: 401, message: 'Token 无效或已过期' });
  }
};

module.exports = authMiddleware;
```

**`controllers/authController.js`**：
```javascript
const prisma = new (require('@prisma/client')).PrismaClient();
const bcrypt = require('bcrypt');
const jwt = require('jsonwebtoken');

const register = async (req, res) => {
  try {
    const { username, password } = req.body;

    // 1. 检查用户名是否已存在
    const existing = await prisma.user.findUnique({
      where: { username }
    });
    if (existing) {
      return res.status(400).json({ code: 400, message: '用户名已存在' });
    }

    // 2. 密码加密
    const hashedPassword = await bcrypt.hash(password, 10);

    // 3. 插入数据库
    const user = await prisma.user.create({
      data: {
        username,
        password: hashedPassword
      }
    });

    // 4. 签发 JWT
    const token = jwt.sign(
      { uid: user.uid },
      process.env.JWT_SECRET,
      { expiresIn: '10h' }
    );

    // 5. 返回响应
    res.status(201).json({
      code: 200,
      message: '注册成功',
      data: { token, uid: user.uid }
    });
  } catch (err) {
    console.error(err);
    res.status(500).json({ code: 500, message: '服务器内部错误' });
  }
};

const login = async (req, res) => {
  try {
    const { username, password } = req.body;

    // 1. 查询用户
    const user = await prisma.user.findUnique({
      where: { username }
    });
    if (!user) {
      return res.status(401).json({ code: 401, message: '用户名或密码错误' });
    }

    // 2. 验证密码
    const match = await bcrypt.compare(password, user.password);
    if (!match) {
      return res.status(401).json({ code: 401, message: '用户名或密码错误' });
    }

    // 3. 签发 JWT
    const token = jwt.sign(
      { uid: user.uid },
      process.env.JWT_SECRET,
      { expiresIn: '10h' }
    );

    // 4. 返回响应
    res.json({
      code: 200,
      message: '登录成功',
      data: { token, uid: user.uid }
    });
  } catch (err) {
    console.error(err);
    res.status(500).json({ code: 500, message: '服务器内部错误' });
  }
};

module.exports = { login, register };
```

**`routes/video.js`**：

**`controllers/videoController.js`**：
```javascript
const prisma = new (require('@prisma/client')).PrismaClient();

const getVideoList = async (req, res) => {
  try {
    const { sort, page = 1, element = 16, q } = req.query;

    // 1. 构建查询条件
    let where = {};
    if (q) {
      where = {
        OR: [
          { title: { contains: q } },
          { uploader: { username: { contains: q } } }
        ]
      };
    }

    // 2. 排序
    let orderBy = {};
    if (sort) {
      const field = sort.startsWith('-') ? sort.slice(1) : sort;
      const direction = sort.startsWith('-') ? 'desc' : 'asc';
      orderBy = { [field]: direction };
    } else {
      orderBy = { createdAt: 'desc' };
    }

    // 3. 分页
    const skip = (page - 1) * element;
    const take = Number(element);

    // 4. 执行查询
    const videos = await prisma.video.findMany({
      where,
      orderBy,
      skip,
      take,
      include: {
        uploader: {
          select: { uid: true, username: true, profilePictureUrl: true }
        }
      }
    });

    // 5. 查询总数
    const total = await prisma.video.count({ where });

    // 6. 返回响应
    res.json({
      code: 200,
      message: 'ok',
      data: videos,
      total
    });
  } catch (err) {
    console.error(err);
    res.status(500).json({ code: 500, message: '服务器内部错误' });
  }
};

const getVideo = async (req, res) => {
  try {
    const { vid } = req.params;

    // 1. 查询视频详情
    const video = await prisma.video.findUnique({
      where: { vid: Number(vid) },
      include: {
        uploader: true,
        videoTags: {
          include: {
            tag: true
          }
        }
      }
    });

    if (!video) {
      return res.status(404).json({ code: 404, message: '视频不存在' });
    }

    // 2. 如果已登录，添加到历史记录
    if (req.user) {
      await prisma.userHistory.upsert({
        where: {
          uid_type_item_id: {
            uid: req.user.uid,
            type: 'video',
            item_id: Number(vid)
          }
        },
        update: { addDate: new Date() },
        create: {
          uid: req.user.uid,
          type: 'video',
          item_id: Number(vid)
        }
      });
    }

    // 3. 返回响应
    res.json({
      code: 200,
      message: 'ok',
      data: video
    });
  } catch (err) {
    console.error(err);
    res.status(500).json({ code: 500, message: '服务器内部错误' });
  }
};

module.exports = { getVideoList, getVideo };
```
