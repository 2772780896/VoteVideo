## 二、核心技术栈

### 1. Express 框架

来源笔记：`17.Express-核心框架.md`

**为什么用 Express**（对比原生 Node.js `http` 模块）：

| 功能 | 原生 Node.js (http 模块) | Express 框架 |
| :--- | :--- | :--- |
| **路由处理** | 需手动解析 `req.url` 并写大量 `if/else` | `app.get()`, `app.post()` 语义化分发 |
| **参数解析** | 需监听 `data` 事件拼字符串，手动 `JSON.parse` | `req.query`, `req.body` 直接拿取对象 |
| **中间件** | 无法原生支持，逻辑全部堆在一起 | 插件化机制，逻辑解耦（如日志、校验、跨域） |
| **静态资源** | 需手动读取文件并设置 MIME 类型 | `app.use(express.static('public'))` 一行搞定 |
| **响应处理** | 只有 `res.end()`，需手动写 Header | `res.send()`, `res.json()`, `res.status()` 链式操作 |

**标准入口 (`app.js`)**：

```javascript
const express = require('express');
const cors = require('cors');
const app = express();

// --- 1. 配置全局中间件 ---
app.use(cors());              // 跨域
app.use(express.json());       // 解析 JSON 请求体

// --- 2. 业务路由 ---
app.post('/api/register', (req, res) => {
  // 逻辑处理...
  res.status(201).json({ msg: '注册成功' });
});

// --- 3. 统一错误处理中间件 ---
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('服务器出错了！');
});

app.listen(3000, () => console.log('启动在 3000'));
```

### 2. JWT 认证

来源笔记：`16.node-全栈手册.md` → 第六章第 4 节 身份鉴权 (JWT 模式)

**核心包**：`npm i jsonwebtoken`

**JWT 三段式原理**：
1. **Header**：算法信息
2. **Payload**：用户信息（Base64 编码，**不可放敏感数据**）
3. **Signature**：签名（Header + Payload + 密钥加密），防止篡改

**JWT 交互流程**（对照 VoteVideo 前端）：
1. 登录成功 → 后端 `jwt.sign()` 发放 Token
2. 前端存储 Token → 之后请求放入 Cookie 或 Header
3. 后端拦截器 → `jwt.verify()` 校验密钥

**登录接口实现示例**：

```javascript
const jwt = require('jsonwebtoken');
const secretKey = 'your-secret-key';  // 放 .env 不要硬编码

app.post('/api/login', (req, res) => {
  const { username, password } = req.body;
  // 查数据库验证...
  const user = { uid: 1, username: 'admin' };
  const token = jwt.sign(user, secretKey, { expiresIn: '10h' });
  res.json({ code: 200, data: { token, uid: user.uid } });
});
```

**鉴权中间件** (`middlewares/auth.js`)：

```javascript
const jwt = require('jsonwebtoken');
const secretKey = process.env.JWT_SECRET;

const authMiddleware = (req, res, next) => {
  // VoteVideo 前端把 token 放在 body.authorization（拦截器逻辑）
  const token = req.body?.authorization || req.query?.authorization;
  if (!token) return res.status(401).json({ code: 401, message: '请登录' });
  
  try {
    const decoded = jwt.verify(token, secretKey);
    req.user = decoded;  // 附加到 req 对象，后续路由可访问
    next();
  } catch (err) {
    res.status(401).json({ code: 401, message: 'Token 无效' });
  }
};

module.exports = authMiddleware;
```

### 3. 数据库交互

来源笔记：`16.node-全栈手册.md` → 第六章第 1 节 数据库交互

**核心包**：`npm i mysql2`

**为什么要用连接池？** 频繁创建和销毁连接非常耗性能。连接池可以复用连接，提高效率。

**连接池配置** (`config/db.js`)：

```javascript
const mysql = require('mysql2/promise');

// 创建连接池
const pool = mysql.createPool({
  host: 'localhost',
  user: 'root',
  password: process.env.DB_PASS,
  database: 'votevideo',
  waitForConnections: true,
  connectionLimit: 10  // 最大连接数
});

module.exports = pool;
```

**在 Controller 中使用**：

```javascript
const db = require('../config/db');

const getVideoList = async (req, res) => {
  const [rows] = await db.query('SELECT * FROM videos WHERE ...');
  res.json({ code: 200, data: rows, total: rows.length });
};
```

### 4. 文件上传

来源笔记：`15.文件传输.md` + `09.Upload页面.md`

**核心包**：`npm i multer`

**前端上传 API**（已定义在 `src/apis/content.js`）：

```javascript
const uploadContent = (type, data) =>
  request.post(`/api/upload/${type}`, data, { needToken: true })
```

**后端实现** (`routes/upload.js`)：

```javascript
const multer = require('multer');
const upload = multer({ dest: 'uploads/' });

app.post('/api/upload/video', authMiddleware, upload.single('video'), (req, res) => {
  // req.file 是视频文件
  // req.body 包含 title, description, cover
});
```

**拦截器 Bug 修复记录**（来自 `09.Upload页面.md`）：
> 旧拦截器把 `config.data` 替换为 `{ authorization: token }`，覆盖上传的 `{ title, cover, ... }`。
> 修复：展开合并 `config.data = { ...(config.data || {}), authorization: token }`。

## 三、前端 API 与后端实现对照表

来源：前端文档 `11.API与数据格式.md`

### 通用要求

- **响应格式统一**：`{ code: 200, message: 'ok', data: T, total?: number }`
- **Token 机制**：`config.needToken === true` 时，前端把 token 放在 `POST body.authorization`
- **拦截器逻辑**：非 DELETE 请求会改 method 为 POST，token 塞进 data

### 路由实现清单

## 四、VoteVideo 前端特殊点（需注意）

### 1. Token 传递方式特殊

来自：`07.Profile页面.md` → 2.2 认证体系

- 非 DELETE 请求：`POST body.authorization`
- DELETE 请求：`data.authorization`（axios 的 data 配置）
- **不是标准的 `Authorization` Header**

### 2. 拦截器会改 method

来自：`11.API与数据格式.md` → 1.1 axios 实例

- GET 请求带 `needToken: true` 时，会被改成 POST
- 后端所有需要鉴权的接口都要用 `app.post()` 或同时支持 GET/POST

### 3. 响应格式必须统一

```javascript
res.json({ code: 200, message: 'ok', data: {...}, total: 100 })
```

- `code: 200` = 成功
- `code: 401` = 未认证
- `code: 404` = 未找到

## 五、数据库表结构设计（参考）

根据前端数据结构推导：

### users 表
```sql
CREATE TABLE users (
  uid INT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  profilePictureUrl VARCHAR(255),
  followerCount INT DEFAULT 0,
  followingCount INT DEFAULT 0
);
```

### videos 表
```sql
CREATE TABLE videos (
  vid INT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(100) NOT NULL,
  coverUrl VARCHAR(255),
  videoUrl VARCHAR(255),
  duration VARCHAR(10),
  viewCount INT DEFAULT 0,
  messageCount INT DEFAULT 0,
  date DATETIME DEFAULT CURRENT_TIMESTAMP,
  uploader_uid INT,
  FOREIGN KEY (uploader_uid) REFERENCES users(uid)
);
```

### comments 表（含自引用实现子评论）
```sql
CREATE TABLE comments (
  cid INT PRIMARY KEY AUTO_INCREMENT,
  vid INT,
  uploader_uid INT,
  text TEXT,
  type ENUM('text', 'picture'),
  viewCount INT DEFAULT 0,
  likeCount INT DEFAULT 0,
  subCommentCount INT DEFAULT 0,
  replyTo_uid INT NULL,
  date DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (vid) REFERENCES videos(vid),
  FOREIGN KEY (uploader_uid) REFERENCES users(uid)
);
```

## 六、待解决问题清单

1. **数据库选型**：MySQL 还是 MongoDB？（关系型数据推荐 MySQL）
2. **文件存储**：视频/图片存本地还是云存储（如七牛云、阿里云 OSS）？
3. **Rerank 模型 404**：`BAAI/bge-reranker-small` 在 hf-mirror.com 不可用，需更换模型或关闭
4. **JWT Secret 管理**：必须放 `.env`，不能硬编码
5. **密码加密**：用户密码必须用 `bcrypt` 哈希存储，不能明文
