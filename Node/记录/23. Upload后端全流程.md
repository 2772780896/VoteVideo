# Upload 后端全流程

## 请求链路总览

```
HTTP 请求进入（multipart/form-data 或 JSON）
  │
  ▼
app.js ──── express.json() + express.urlencoded()
  │         + express.static('/uploads')  ← 让已上传文件可通过 HTTP 直接访问
  │
  ▼
routes/uploadRoutes.js ──── 路由映射 + needToken + multer 中间件
  │                          （视频/动态挂 multer，文章不挂）
  │
  ▼
middleware/uploadMiddleware.js ──── multer 解析二进制流 → 文件写入磁盘 → req.files 注入
  │                                  （仅在 /video 和 /post 路由生效）
  │
  ▼
controllers/uploadController.js ── 从 req.user 取 uid、req.files 取文件路径、req.body 取文本字段
  │                                 toUrl() 把磁盘路径转成可访问的相对 URL
  │
  ▼
services/uploadService.js ── 参数校验 → prisma.create 写入数据库
  │
  ▼
utils/response.js ── sendSuccess / sendError
  │
  ▼
HTTP 响应返回（{ code: 201, data: { vid/eid/pid } }）
```

> 上传模块是**独立体系**，不走 baseService / transformer / interactionService。
> 创建类接口统一归入 uploadService，不放在各媒体 controller 中。

## 共享机制

### 强制认证 (needToken)

`middleware/authMiddleware.js`

- 所有上传接口都必须登录，使用 `needToken`（不是 `optionalAuth`）
- 验证 `Authorization: Bearer <token>` → 有效则 `req.user = { uid, username, iat, exp }`
- 无效或缺失 → 返回 401，**阻断请求**

### 文件上传中间件 (multer)

`middleware/uploadMiddleware.js`

**职责**：拦截 `multipart/form-data` 请求，把二进制文件写入磁盘，把文件信息注入 `req.files`

**存储引擎**：`multer.diskStorage()`，两步决定文件落盘位置：
1. `destination` → 根据表单字段名分目录（video → `uploads/video/`、cover → `uploads/cover/`、images → `uploads/post/`）
2. `filename` → `Date.now() + 随机数 + 原始扩展名`，保证不重名

**两个中间件实例**：

| 中间件 | API | req.files 结构 | 适用路由 |
|--------|-----|---------------|---------|
| `videoUpload` | `multer.fields([{name:'video'}, {name:'cover'}])` | `{ video: [file], cover: [file] }`（对象） | `/video` |
| `postUpload` | `multer.array('images', 9)` | `[file0, file1, ...]`（数组） | `/post` |

**安全机制**：
- `fileFilter`：白名单 mimetype（视频 mp4/webm/mov、图片 jpg/png/gif/webp），不匹配直接抛错
- `limits.fileSize`：视频 ≤100MB，图片 ≤5MB

**启动时自动建目录**：`fs.mkdirSync(dir, { recursive: true })` 预创建 `uploads/video`、`uploads/cover`、`uploads/post`

### 静态文件服务 (express.static)

`app.js`

```js
app.use('/uploads', express.static(path.join(__dirname, 'uploads')))
```

- multer 把文件写入磁盘 → express.static 让磁盘文件可通过 HTTP 直接访问
- 例：磁盘路径 `uploads/video/1718500000000-837291645.mp4` → URL `http://localhost:3000/uploads/video/1718500000000-837291645.mp4`

### 路径转换 (toUrl)

`controllers/uploadController.js`

```js
const toUrl = (file) => {
  if (!file) return null
  const dir = file.destination.replace(/\\/g, '/')  // Windows 反斜杠 → URL 正斜杠
  return `/${dir}/${file.filename}`
}
```

- multer file 对象包含 `{ destination, filename, path, mimetype, size, ... }`
- `toUrl()` 把 `destination` + `filename` 拼成前端可用的相对 URL：`/uploads/video/xxx.mp4`
- 前端拼接 `baseURL` 后即可访问：`http://localhost:3000/uploads/video/xxx.mp4`

### 统一响应 (utils/response.js)

| 函数 | 用途 | 输出 |
|------|------|------|
| `sendSuccess(res, data, msg, 201)` | 上传成功 | `{ code: 201, message, data }` |
| `sendError(res, error, msg)` | 错误 | 业务错误用 `error.statusCode`，否则 500 |

> 上传模块不用 `sendList`（不是列表接口）。

### 统一错误处理

- service 层参数校验不通过 → `throw new Error(msg)` + `err.statusCode = 400`
- multer 文件校验不通过 → multer 直接抛错，不进 controller
- controller 用 `try/catch` 捕获 → `sendError(res, error, '描述')`
- `sendError` 根据 `error.statusCode` 区分业务错误（400）和服务器错误（500）

## API 1：上传视频

```
POST /api/upload/video
Content-Type: multipart/form-data（axios 自动设置）
Header: Authorization: Bearer <token>
超时: 120000ms
```

| FormData 字段 | 类型 | 必填 | 说明 |
|--------------|------|------|------|
| `video` | File | 是 | mp4/webm/mov，≤100MB |
| `cover` | File | 否 | jpg/png/gif/webp，≤5MB（canvas 截图或本地上传） |
| `title` | string | 是 | 视频标题 |
| `description` | string | 否 | 视频描述 |

**路由处理链**：`needToken` → `videoUpload`（multer.fields） → `uploadController.uploadVideo`

**Controller 处理流程**：
1. 从 `req.user` 取 `uid`
2. 从 `req.body` 取 `title`、`description`（multer 解析 multipart 时，文本字段自动写入 `req.body`）
3. 从 `req.files.video[0]` 取视频文件对象 → `toUrl()` 转 URL
4. 从 `req.files.cover[0]` 取封面文件对象 → `toUrl()` 转 URL（可选）
5. 调用 `uploadService.uploadVideo(uid, { title, description, coverUrl, videoUrl })`
6. `sendSuccess(res, result, '上传成功', 201)`

**Service 处理流程**：
1. 参数校验：`!title || !videoUrl` → 抛 400 错误
2. `prisma.video.create({ data: { title, coverUrl, videoUrl, duration: null, viewCount: 0, commentCount: 0, likeCount: 0, favouriteCount: 0, uploader_uid: uid } })`
3. 返回 `{ vid: newVideo.vid }`

**数据库写入（Video 表）**：

| 字段 | 值 | 说明 |
|------|----|------|
| `title` | 前端传入 | 视频标题 |
| `coverUrl` | `/uploads/cover/xxx.jpg` 或 null | 封面 URL |
| `videoUrl` | `/uploads/video/xxx.mp4` | 视频 URL |
| `duration` | null | TODO：后续用 ffprobe 解析 |
| `uploader_uid` | uid | 上传者 |
| 计数字段 | 0 | viewCount / commentCount / likeCount / favouriteCount |

## API 2：上传文章

```
POST /api/upload/essay
Content-Type: application/json
Header: Authorization: Bearer <token>
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 文章标题 |
| `description` | string | 是 | 文章内容 |

**路由处理链**：`needToken` → `uploadController.uploadEssay`（**无 multer 中间件，纯 JSON**）

**Controller 处理流程**：
1. 从 `req.user` 取 `uid`
2. 从 `req.body` 取 `title`、`description`
3. 调用 `uploadService.uploadEssay(uid, { title, description })`
4. `sendSuccess(res, result, '发布成功', 201)`

**Service 处理流程**：
1. 参数校验：`!title || !description` → 抛 400 错误
2. `prisma.essay.create({ data: { title, text: description, viewCount: 0, commentCount: 0, likeCount: 0, favouriteCount: 0, uploader_uid: uid } })`
3. 返回 `{ eid: newEssay.eid }`

> 文章不涉及文件上传，所以路由不挂 multer 中间件，请求体走标准 JSON 解析。

## API 3：上传动态

```
POST /api/upload/post
Content-Type: multipart/form-data（axios 自动设置）
Header: Authorization: Bearer <token>
超时: 120000ms
```

| FormData 字段 | 类型 | 必填 | 说明 |
|--------------|------|------|------|
| `text` | string | 否 | 动态文本（≤500 字符） |
| `images` | File[] | 否 | 图片文件（最多 9 张，单张 ≤5MB，jpg/png/gif/webp） |

`text` 和 `images` 至少提供一个。

**路由处理链**：`needToken` → `postUpload`（multer.array） → `uploadController.uploadPost`

**Controller 处理流程**：
1. 从 `req.user` 取 `uid`
2. 从 `req.body` 取 `text`
3. 从 `req.files`（数组）→ `.map(toUrl)` 转 URL 数组
4. 调用 `uploadService.uploadPost(uid, { text, imageUrls })`
5. `sendSuccess(res, result, '发布成功', 201)`

**Service 处理流程**：
1. 参数校验：`!text && (!imageUrls || imageUrls.length === 0)` → 抛 400 错误
2. 图片 URL 数组 → `JSON.stringify(imageUrls)` 序列化（SQLite 不支持原生数组类型）
3. `prisma.post.create({ data: { text, pictureList: JSON字符串, videoList: null, viewCount: 0, commentCount: 0, likeCount: 0, uploader_uid: uid } })`
4. 返回 `{ pid: newPost.pid }`

**pictureList 存储格式**：

```json
["/uploads/post/xxx.jpg", "/uploads/post/yyy.png"]
```

前端读取时 `JSON.parse(pictureList)` 还原为数组。

## 前端数据流（完整链路）

```
前端用户选择文件
  │
  ├─ 视频：<input type="file"> → File 对象存 state
  │         URL.createObjectURL(file) → blob URL 仅用于 <video> 预览
  │
  ├─ 封面：canvas.toBlob(callback) → Blob → new File([blob], 'cover.jpg')
  │      或 <input type="file"> → File 对象存 state
  │         URL.createObjectURL(file) → blob URL 仅用于 <img> 预览
  │
  └─ 图片：<input type="file" multiple> → File[] 存 state
           每项 { file: File, preview: blob URL }
  │
  ▼
构建 FormData
  │  formData.append('video', videoFile)
  │  formData.append('cover', coverFile)      // 可选
  │  formData.append('title', title)
  │  formData.append('description', description)
  │
  ▼
uploadContent(type, formData)
  │  → request.post('/api/upload/{type}', formData, { needToken: true, timeout: 120000 })
  │  → axios 自动设置 Content-Type: multipart/form-data（含 boundary）
  │
  ▼
后端接收（multer 解析二进制流 → 文件落盘 → req.files 注入）
  │
  ▼
controller → toUrl() → service → prisma.create → 返回 { vid/eid/pid }
  │
  ▼
前端通过 /uploads/xxx 访问文件（express.static 提供服务）
```

## 与五媒体流程的差异

| 维度 | 五媒体（video/essay/post/tag/comment） | 上传模块 |
|------|---------------------------------------|---------|
| 中间件 | `optionalAuth`（可选认证） | `needToken`（强制认证） |
| 通用服务 | `baseService`（分页/排序/推荐） | 无（直接 `prisma.create`） |
| 数据转换 | `transformer`（纯函数） | 无（controller 内 `toUrl()`） |
| 交互状态 | `interactionService.mergeInteractions` | 无（新建记录无交互） |
| 响应格式 | `sendList`（列表）/ `sendSuccess`（详情） | 仅 `sendSuccess`（201） |
| 请求格式 | JSON (`application/json`) | FormData (`multipart/form-data`) 或 JSON |
| 文件处理 | 无 | multer 存盘 + express.static 服务 |

## 文件索引

| 层级 | 文件 | 职责 |
|------|------|------|
| 入口 | `app.js` | express.static('/uploads') 静态文件服务 |
| 路由注册 | `routes/index.js` | `{ path: '/api/upload', router: uploadRoutes }` |
| 路由 | `routes/uploadRoutes.js` | URL 映射 + needToken + multer 中间件挂载 |
| 中间件 | `middleware/uploadMiddleware.js` | multer 配置（存储引擎/类型过滤/体积限制/建目录） |
| 中间件 | `middleware/authMiddleware.js` | needToken JWT 验证 |
| 控制器 | `controllers/uploadController.js` | 参数提取 + toUrl() 路径转换 + 调 service + response 输出 |
| 服务 | `services/uploadService.js` | 参数校验 + prisma.create 写入数据库 |
| 响应 | `utils/response.js` | 统一响应格式（sendSuccess / sendError） |
| 数据库 | `prisma/schema.prisma` | Video / Essay / Post 表定义 |
