## 详细变更说明

### 1. 数据库模型扩展 — `prisma/schema.prisma`

**解决的问题：** 缺少转发和踩功能的数据模型；Tag 缺少 viewCount；Post 缺少 favouriteCount/reshareCount

**具体变更：**

- **Tag 模型** 新增 `viewCount Int @default(0)` 字段
- **Video 模型** 新增 `reshareCount Int @default(0)` 字段
- **Essay 模型** 新增 `reshareCount Int @default(0)` 字段
- **Post 模型** 新增 `reshareCount Int @default(0)` 和 `favouriteCount Int @default(0)` 字段
- **Comment 模型** 新增 `dislikeCount Int @default(0)` 字段
- **新增 UserReshare 模型** — 用户转发关联表，复合主键 `[uid, type, item_id]`，关联 User 表
- **新增 UserDislike 模型** — 用户踩关联表，复合主键 `[uid, type, item_id]`，关联 User 表
- **User 模型** 新增 `reshares UserReshare[]` 和 `dislikes UserDislike[]` 关联

---

### 2. 私信接口参数修复 — `messageService.js` + `messageController.js`

**解决的问题：** ❌ 严重问题1 — 前端发送 `{ dialogueMid, text }` 但后端期望 `{ receiverUid, text }`

**messageController.js 变更：**
- `sendMessage` 处理器现在从 `req.body` 中同时解构 `dialogueMid` 和 `receiverUid`
- 将 `dialogueMid` 作为第四个参数传递给 `messageService.sendMessage`

**messageService.js 变更：**
- `sendMessage` 函数签名改为 `sendMessage(senderUid, receiverUid, text, dialogueMid = null)`
- 新增优先通过 `dialogueMid` 查找已有对话的逻辑
- 保留原有的通过 `receiverUid` 查找或创建对话的兜底逻辑
- SSE 推送接收者改为从对话参与者中自动推断（而非依赖 `receiverUid` 参数）

---

### 3. 对话/通知列表路径修复 — `userController.js`

**解决的问题：** ❌ 严重问题2 — 前端请求 `/api/user/profile/message/dialogueList` 但后端路由是 `/api/user/message/dialogues`

**具体变更：**
- `getProfileSubdata` 新增 `profileType === 'message'` 分支
- `dataType === 'dialogueList'` 时委托 `messageService.getDialogues(uid)` 获取对话列表
- `dataType === 'notificationList'` 时委托 `messageService.getNotifications(uid)` 获取通知列表
- 前端通过原有的 `getProfileSubdata` 接口即可获取消息数据，无需修改调用路径

---

### 4. 关注列表接口补全 — `userController.js`

**解决的问题：** ❌ 严重问题3 — 后端 `getProfileSubdata` 不支持 `follow/followingList`

**具体变更：**
- `getProfileSubdata` 新增 `profileType === 'follow'` 分支
- `dataType === 'followingList'` 时查询 `UserFollowing` 表，关联查询被关注者的 `uid`、`username`、`profilePictureUrl`、`followerCount`
- 返回格式与前端 `ProfileFollow.jsx` 期望一致

---

### 5. 轮播图响应格式修复 — `videoController.js`

**解决的问题：** ❌ 严重问题4 — 后端直接返回数组，前端期望 `{ items: [...] }`

**具体变更：**
- `getCarousel` 的响应从 `sendSuccessResponse(res, carouselItems)` 改为 `sendSuccessResponse(res, { items: carouselItems })`
- 前端 `useData` 解包后 `data.items` 可正常访问

---

### 6. 收藏/历史子数据格式修复 — `userController.js`

**解决的问题：** ❌ 严重问题5 — 收藏和历史只返回原始关联记录，没有关联查询实际内容

**具体变更：**
- `favourites` 分支重写：先查 `UserFavourite` 获取 `item_id` 列表，再根据 `dataType` 分别查询 Video/Post/Essay 的完整数据（包含 uploader 信息）
- `history` 分支重写：同上逻辑，查询 `UserHistory` 后关联实际内容
- 返回格式与 `uploads` 分支一致，前端可用相同的 Card 组件渲染

---

### 7. uid 查询参数支持 — `baseService.js`

**解决的问题：** ❌ 严重问题6 — 用户主页通过 `?uid=xxx` 筛选投稿，后端不处理

**具体变更：**
- `getListData` 方法新增 `options.uid` 判断
- 当传入 `uid` 时，自动添加 `where.uploader_uid = parseInt(options.uid)` 条件
- 影响范围：Video、Essay、Post 的列表接口均自动支持 uid 筛选

---

### 8. 字段名与数据格式统一修复

#### 8a. `authController.js` — username → userName
- `getProfile` 返回的 data 从直接返回 Prisma 原始对象改为手动映射
- `username` 字段重命名为 `userName`，与前端 `Profile.jsx` 和 `UserMenu.jsx` 匹配

#### 8b. `userController.js` — getUserDetail 补全 info 字段
- 响应对象新增 `info: user.info`
- 前端 `User.jsx` 的 `userData.info` 不再为 undefined

#### 8c. `commentService.js` — 评论字段别名
- 新增 `commenter` 字段作为 `uploader` 的别名（前端 `CommentCard.jsx` 使用 `commenter`）
- 新增 `subComments` 字段作为 `subCommentList` 的别名（前端使用 `subComments`）
- 新增 `isLiked: false` 和 `isDisliked: false` 初始值
- 新增 `dislikeCount` 格式化字段

#### 8d. 计数类型修复 — videoService / essayService / postService
- `likeCount`、`favouriteCount`、`reshareCount` 改为返回原始数字（不再格式化为字符串）
- 前端 `InteractionBar` 可以对这些值做 `+1`/`-1` 的乐观更新运算
- `viewCount`、`commentCount` 仍保留 `formatCount()` 格式化（前端仅用于展示）

#### 8e. videoService — uploader.isFollowing
- `transformVideoData` 中 uploader 对象新增 `isFollowing` 字段
- 通过检查 `video.uploader.followers` 中是否包含 `currentUid` 来计算

#### 8f. videoService — isReshared
- `transformVideoData` 新增 `isReshared` 字段
- 通过检查 `video.reshares` 中是否包含 `currentUid` 来计算

#### 8g. tagService — viewCount
- `transformTagData` 新增 `viewCount` 字段（原始数字）

---

### 9. 新增功能：转发（reshare）和踩（dislike）

#### 9a. interactService.js
- `RESOURCE_CONFIG` 所有条目新增 `hasReshare` 和 `hasDislike` 配置项
  - video/essay/post: `hasReshare: true, hasDislike: false`
  - comment: `hasReshare: false, hasDislike: true`
  - tag: `hasReshare: false, hasDislike: false`
- 新增 `toggleReshare(uid, type, id, method)` — 操作 `UserReshare` 表并更新 `reshareCount`
- 新增 `toggleDislike(uid, type, id, method)` — 操作 `UserDislike` 表并更新 `dislikeCount`

#### 9b. interactController.js
- 新增 `reshare` 处理器 — 完整的错误处理（P2002/P2025/不存在/不支持）
- 新增 `dislike` 处理器 — 返回 `{ dislikeCount }` 用于前端乐观更新

#### 9c. interactRoutes.js
- 新增 `POST /:type/:id/reshare` 和 `DELETE /:type/:id/reshare` 路由
- 新增 `POST /:type/:id/dislike` 和 `DELETE /:type/:id/dislike` 路由
- 均挂载 `needToken` 中间件

---

### 10. 消息服务修复 — `messageService.js`

#### 10a. getNotifications 修复
- 查询条件从 `from_uid: uid`（自己发出的通知）改为 `to_uid: uid`（发给自己的通知）
- 新增 `include: { fromUser }` 关联查询发送者信息
- 返回格式从 `{ nid, from: { userName: '', profilePictureUrl: '' } }` 改为 `{ mid, sender: { userName, profilePictureUrl } }`，匹配前端 `ProfileMessage.jsx` 期望

#### 10b. userRoutes.js — needToken 中间件
- `getProfileSubdata` 路由新增 `needToken` 中间件
- 确保未登录用户无法访问个人资料子数据
