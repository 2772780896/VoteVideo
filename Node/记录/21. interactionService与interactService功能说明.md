# 21. interactionService 与 interactService 功能说明

## 两个文件的关系

```
interactionService.js          interactService.js
       读                              写
  "有没有赞过？"               "执行点赞动作"
       │                              │
       └──── 共享 INTERACTION_CONFIG ────┘
```

| | interactionService | interactService |
|--|-------------------|-----------------|
| 职责 | 查询交互状态（只读） | 执行交互动作（写入） |
| 改数据库？ | 不改 | 改（插入/删除/计数±1） |
| 调用时机 | 每次加载列表/详情页 | 用户点击点赞/收藏/转发按钮 |
| 调用频率 | 高（每次页面加载） | 低（用户主动操作） |
| 被谁调 | baseService 自动调、各 Service 详情页 | interactController |

**为什么不合并？** 读和写是两个不同的关注点。读操作是高频批量的（一次查100条的点赞状态），写操作是低频单条的（改两张表）。合并后文件约550行，职责混淆。保持分开 + 共享配置表是当前最优结构。

---

## INTERACTION_CONFIG — 统一配置表

定义在 `interactionService.js` 中，被 `interactService.js` 导入。一张表同时服务两个模块：

| 类型 | model | idField | name | likes | favourites | reshares | dislikes | hasHistory | hasReply |
|------|-------|---------|------|-------|------------|----------|----------|------------|----------|
| video | video | vid | 视频 | ✓ | ✓ | ✓ | ✗ | ✓ | ✓ |
| essay | essay | eid | 文章 | ✓ | ✓ | ✓ | ✗ | ✓ | ✓ |
| post | post | pid | 动态 | ✓ | ✓ | ✓ | ✗ | ✓ | ✓ |
| comment | comment | cid | 评论 | ✓ | ✓ | ✗ | ✓ | ✗ | ✓ |
| tag | tag | tid | 标签 | ✓ | ✓ | ✗ | ✓ | ✗ | ✓ |

**字段分组：**
- `model / idField / name`：资源定位（interactService 用来查表和报错提示）
- `likes / favourites / reshares / dislikes`：交互查询开关（interactionService 用来决定查哪些表）
- `hasHistory / hasReply`：功能开关（interactService 用来判断是否支持历史记录和回复）

## interactionService.js — 读交互状态

### checkInteractions(type, ids, currentUid)

批量查询用户对一组资源的交互状态。

**执行流程：**
```
1. 根据 type 从配置表读取支持哪些交互
2. 对每种支持的交互，并行查对应的表（UserLike / UserFavourite / UserReshare / UserDislike）
3. 将查出的 ID 数组转为 Set（O(1) 查找）
4. 为每个 id 构建 { isLiked, isFavourited, isReshared, isDisliked }
5. 返回映射表：{ 123: { isLiked: true, ... }, 456: { ... } }
```

**查的4张交互表：**

| 表名 | 对应操作 | 字段 |
|------|---------|------|
| UserLike | 点赞 | uid + type + item_id |
| UserFavourite | 收藏 | uid + type + item_id |
| UserReshare | 转发 | uid + type + item_id |
| UserDislike | 踩 | uid + type + item_id |

**为什么用 Set？**
数组的 `includes` 是 O(n) 查找，Set 的 `has` 是 O(1) 查找。100条数据时差别不大，数据量大了性能优势明显。

### mergeInteractions(type, items, idField, currentUid)

批量合入交互状态到数据列表（原地修改）。

**执行流程：**
```
1. 从 items 中提取所有 id
2. 调 checkInteractions 查交互状态
3. 用 Object.assign 把 isLiked 等字段合入每条数据
```

**被谁调：**
- `baseService.getListData` 自动调（列表页）
- 各 Service 的 `getXxxDetailData` 手动调（详情页）

## interactService.js — 写交互动作

### 函数清单

| 函数 | 作用 | 涉及的表 |
|------|------|---------|
| `findResource(type, id)` | 查资源是否存在 | 各媒体表 |
| `toggleLike(uid, type, id, method)` | 点赞/取消赞 | UserLike + 媒体表(计数±1) |
| `toggleFavourite(uid, type, id, method)` | 收藏/取消收藏 | UserFavourite + 媒体表(计数±1) |
| `toggleReshare(uid, type, id, method)` | 转发/取消转发 | UserReshare + 媒体表(计数±1) |
| `toggleDislike(uid, type, id, method)` | 踩/取消踩 | UserDislike + 媒体表(计数±1) |
| `toggleFollow(uid, targetUid, method)` | 关注/取消关注 | UserFollowing + User表(粉丝数±1) |
| `recordHistory(uid, type, id)` | 记录浏览历史 | UserHistory |
| `createReply(uid, mediaType, mediaId, text)` | 回复（创建评论） | Comment + 媒体表(评论数+1) |

### toggle 系列函数的通用模式

所有 toggle 函数遵循相同模式：

```
1. findResource(type, id)     → 资源不存在就报错
2. 检查 config 是否支持该操作  → 不支持就报错
3. method === 'POST' ?
   → 是：插入记录 + 计数+1
   → 否：删除记录 + 计数-1
```

### createReply 的特殊逻辑

回复评论时（mediaType === 'comment'）需要：
1. 查被回复评论的父级资源（vid/eid/pid/tid），继承到子评论
2. 增加父评论的 subCommentCount
3. 不增加媒体的 commentCount（因为只增加 subCommentCount）

回复媒体时（mediaType !== 'comment'）需要：
1. 增加媒体的 commentCount

## 数据流全景

### 读流程（加载页面时）

```
前端请求 GET /api/video
  → videoController
  → videoService.getVideoListData (= baseService.getListData)
    → 查数据库拿视频列表
    → transformVideoData 转换格式
    → interactionService.mergeInteractions('video', items, 'vid', uid)
      → checkInteractions 并行查 UserLike + UserFavourite + UserReshare
      → Object.assign 合入 isLiked/isFavourited/isReshared
  → 返回给前端
```

### 写流程（用户点按钮时）

```
前端请求 POST /api/interact/like  { type: 'video', id: 123 }
  → interactController
  → interactService.toggleLike(uid, 'video', 123, 'POST')
    → findResource('video', 123)  → 视频不存在就报错
    → 检查 config.likes → video 支持点赞
    → UserLike.create(...)        → 插入点赞记录
    → video.update(likeCount +1)  → 更新视频点赞数
  → 返回成功
```
