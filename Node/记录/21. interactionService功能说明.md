# 21. interactionService.js 功能说明

## 定位

通用交互状态查询服务 —— **只读**，不改数据库。回答一个问题："当前用户对这些内容赞/藏/转/踩过吗？"

与 `interactService.js` 的区别：

| | interactionService | interactService |
|--|-------------------|-----------------|
| 职责 | 读交互状态 | 写交互动作 |
| 改数据库？ | 不改 | 改（插入/删除/计数±1） |
| 调用时机 | 每次加载页面 | 用户点按钮时 |

---

## INTERACTION_CONFIG — 统一配置表

定义在 `interactionService.js` 中，同时被 `interactService.js` 导入使用。

| 类型 | model | idField | name | likes | favourites | reshares | dislikes | hasHistory | hasReply |
|------|-------|---------|------|-------|------------|----------|----------|------------|----------|
| video | video | vid | 视频 | ✓ | ✓ | ✓ | ✗ | ✓ | ✓ |
| essay | essay | eid | 文章 | ✓ | ✓ | ✓ | ✗ | ✓ | ✓ |
| post | post | pid | 动态 | ✓ | ✓ | ✓ | ✗ | ✓ | ✓ |
| comment | comment | cid | 评论 | ✓ | ✓ | ✗ | ✓ | ✗ | ✓ |
| tag | tag | tid | 标签 | ✓ | ✓ | ✗ | ✓ | ✗ | ✓ |

**字段分组：**
- `model / idField / name`：资源定位（interactService 用来查表和报错提示）
- `likes / favourites / reshares / dislikes`：交互查询开关（interactionService 用来决定查哪些表）
- `hasHistory / hasReply`：功能开关（interactService 用来判断是否支持历史记录和回复）

## checkInteractions(type, ids, currentUid)

批量查询用户对一组资源的交互状态。

### 执行流程

```
1. 根据 type 从配置表读取支持哪些交互
2. 对每种支持的交互，并行查对应的表（UserLike / UserFavourite / UserReshare / UserDislike）
3. 将查出的 ID 数组转为 Set（O(1) 查找）
4. 为每个 id 构建 { isLiked, isFavourited, isReshared, isDisliked }
5. 返回映射表：{ 123: { isLiked: true, ... }, 456: { ... } }
```

### 查的4张交互表

| 表名 | 对应操作 | 联合唯一键 |
|------|---------|-----------|
| UserLike | 点赞 | uid + type + item_id |
| UserFavourite | 收藏 | uid + type + item_id |
| UserReshare | 转发 | uid + type + item_id |
| UserDislike | 踩 | uid + type + item_id |

### 为什么用 Set

数组的 `includes` 是 O(n) 查找，Set 的 `has` 是 O(1) 查找。数据量大了性能优势明显。

### 返回值示例

```
输入: checkInteractions('video', [1, 2, 3], 当前用户ID=5)
输出: {
  1: { isLiked: true,  isFavourited: false, isReshared: false, isDisliked: false },
  2: { isLiked: false, isFavourited: true,  isReshared: false, isDisliked: false },
  3: { isLiked: false, isFavourited: false, isReshared: false, isDisliked: false }
}
```

## mergeInteractions(type, items, idField, currentUid)

批量合入交互状态到数据列表（原地修改）。

### 执行流程

```
1. 从 items 中提取所有 id
2. 调 checkInteractions 查交互状态
3. 用 Object.assign 把 isLiked 等字段合入每条数据
```

### 被谁调

| 调用者 | 场景 |
|--------|------|
| `baseService.getListData` | 列表页自动合入（所有5种媒体） |
| `videoService.getVideoDetailData` | 视频详情页手动合入 |
| `essayService.getEssayDetailData` | 文章详情页手动合入 |
| `postService.getPostDetailData` | 动态详情页手动合入 |
| `commentService.getCommentDetailData` | 评论详情页手动合入 |
| 各 Service 的 `getRelatedXxxData` | 相关推荐手动合入 |

---

## 数据流（读流程）

```
前端请求 GET /api/video
  → videoController
  → videoService.getVideoListData (= baseService.getListData)
    → 查数据库拿视频列表
    → transformVideoData 转换格式
    → mergeInteractions('video', items, 'vid', uid)
      → checkInteractions 并行查 UserLike + UserFavourite + UserReshare
      → Object.assign 合入 isLiked/isFavourited/isReshared
  → 返回给前端（每条视频都带有交互状态）
```
