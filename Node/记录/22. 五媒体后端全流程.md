# 五媒体后端全流程

## 请求链路总览

```
HTTP 请求进入
  │
  ▼
routes/{media}Routes.js ──── 路由映射 + optionalAuth 中间件
  │
  ▼
controllers/{media}Controller.js ── 提取 req.query / req.user，调用 service
  │
  ▼
services/{media}Service.js ── 业务编排（大部分委托 baseService）
  │
  ├─► baseService.js ──── 通用分页/排序/推荐/CRUD
  │     ├─► Prisma（数据库查询）
  │     └─► interactionService.js（交互状态合并）
  │
  ├─► transformers/{media}Transformer.js ── 纯函数数据转换
  │
  ▼
utils/response.js ── sendList / sendSuccess / sendError
  │
  ▼
HTTP 响应返回
```

> 创建类接口统一由 `uploadService` 处理（`POST /api/upload/video`、`/essay`、`/post`），不放在各媒体 controller 中。

## 共享机制

### 可选认证 (optionalAuth)

`middleware/authMiddleware.js`

- 检查 `Authorization: Bearer <token>` 请求头
- 有效 → `req.user = { uid, username, iat, exp }`
- 无效或缺失 → 静默跳过，不阻断请求

### 播放量自动增加 (trackView)

`baseService.js` — `getItemData` 方法

- MODULE_CONFIG 中所有媒体配置了 `trackView: true`
- 详情页查询时自动原子操作 `viewCount + 1`，无需各 service 手动处理

### 数据转换 (transformer)

`services/transformers/{media}Transformer.js`

纯函数，不查数据库。将 Prisma 对象转为前端友好格式：
- 交互状态字段（`isLiked` 等）默认 `false`，后续由 interactionService 覆盖
- `uploader` 统一格式：`{ uid, userName, profilePictureUrl, isFollowing: false }`
- 日期统一 `formatDate()` → `"YYYY-MM-DD"`

### 交互状态合并 (mergeInteractions)

`services/interactionService.js`

1. 提取 items 的 ID 数组
2. 并行查询 UserLike / UserFavourite / UserReshare（多态关联表，无法 Prisma include）
3. `Object.assign` 原地覆盖 transformer 的默认 `false` 值

### 统一响应 (utils/response.js)

| 函数 | 用途 | 输出 |
|------|------|------|
| `sendSuccess(res, data, msg, status)` | 详情/特殊 | `{ code, message, data }` |
| `sendList(res, result)` | 列表 | `{ code: 200, data: { items, total } }` |
| `sendError(res, error, msg)` | 错误 | 业务错误用 `error.statusCode`，否则 500 |

## API 1：列表

```
GET /api/{media}?page=1&element=16&sort=-date&q=关键词
```

| 媒体 | 路由 | 默认排序 | 特殊参数 |
|------|------|---------|---------|
| video | `/` 和 `/main` | `-viewCount` | `following`（关注动态） |
| essay | `/` | `-viewCount` | `following` |
| post | `/` | `-viewCount` | `following` |
| tag | `/` | `-viewCount` | — |
| comment | `/` | `-viewCount` | `vid/eid/pid/tid`（所属资源过滤） |

**Controller 处理流程**（五媒体统一）：
1. 从 `req.query` 提取 `page, element, sort, q, currentUid`
2. 调用 `{media}Service.getListData(options)` → `sendList(res, result)`

**Service 处理流程**：
- **video/essay/post/tag**：直接委托 `baseService.getListData(options)`
- **comment**：额外传入 `where: { replyTo_cid: null }`（只取顶级评论）+ 自定义 `include`

**baseService.getListData 内部**：
1. `parsePagination(page, element)` → `{ skip, take }`
2. `parseSortParam(sort)` → 排序对象
3. 构建 where：搜索 `q` → `searchField contains`；`following` → 先查关注列表
4. `Promise.all` 并行 `findMany` + `count`
5. 逐条调 transformer
6. 若有 `currentUid`，调 `mergeInteractions` 覆盖交互状态

**comment 的特殊之处**：
- 排序参数需转换：前端传 `1` → `-likeCount`，`-1` → `likeCount`
- 接受 `vid/eid/pid/tid` 过滤评论所属资源

## API 2：详情

```
GET /api/{media}/:id
```

| 媒体 | 路径参数 | 播放量 +1 |
|------|---------|----------|
| video | `:vid` | ✓（trackView） |
| essay | `:eid` | ✓（trackView） |
| post | `:pid` | ✓（trackView） |
| tag | `:tid` | ✓（trackView） |
| comment | `:cid` | ✓（trackView） |

**Controller 处理流程**（五媒体统一）：
1. 从 `req.params` 提取 ID
2. `currentUid = req.user?.uid || null`
3. 调用 `{media}Service.getDetailData(id, currentUid)` → `sendSuccess(res, item)`

**Service 处理流程**（五媒体统一）：
1. `baseService.getItemData(id, { throwIfNotFound: true })` — 不存在时抛 404，**trackView 已开启则自动 +1 播放量**
2. 调 transformer（video 详情页额外 `includeVideoUrl: true, includeTags: true`）
3. 若有 `currentUid`，调 `mergeInteractions`

**路由顺序注意**：`/:id` 必须在 `/related` 之后定义，否则 `related` 会被路径参数误匹配。

## API 3：推荐

```
GET /api/{media}/related?id=xxx&page=1&element=5
```

| 媒体 | 推荐策略 | 说明 |
|------|---------|------|
| video | **tag 匹配** | 查当前视频的 tag → 找共同 tag 的视频 → 按匹配数排序 |
| essay | 取最新 | 排除自己，按日期降序 |
| post | 取最新 | 排除自己，按日期降序 |
| tag | 取最热 | 排除自己，按 `-viewCount` 降序 |
| comment | — | **无推荐接口** |

**Controller 处理流程**（四媒体统一）：
1. 提取 `id, sort, page, element, currentUid`
2. 调用 `{media}Service.getRelatedData(options)` → `sendList(res, result)`

**Service 处理流程**：委托 `baseService.getRelatedData(options)`。

**baseService.getRelatedData 内部**：

```
有 tagRelationField 的模型（仅 video）：
  1. 查当前资源的 tag（通过 videoTags 中间表）
  2. 查中间表中有相同 tag 的其他资源（排除自己）
  3. 统计共同 tag 数 → 降序排列 → 取前 N 个
  4. 按排名顺序查完整数据
  5. 无 tag 时回退为默认排序

无 tagRelationField 的模型（essay/post/tag）：
  排除自己 → 按默认排序取最新
```

**video 特殊**：推荐列表覆盖 transformFunction 为 `includeTags: false`（推荐卡片不需要标签）。

## API 4：轮播图（仅 Video）

```
GET /api/picture/carousel?number=5
```

不经过 baseService：
- 按 `viewCount` 降序取前 N 条，仅选 `vid, coverUrl, title`
- `transformToCarouselItems` → `[{ id: vid, src: coverUrl }]`
- 不涉及交互状态

## Message（独立体系，建议单独文档）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/stream` | GET | SSE 实时消息流 |
| `/send` | POST | 发送私信 |
| `/dialogues` | GET | 对话列表 |
| `/notifications` | GET | 通知列表 |

- 使用 `needToken`（强制认证），不是 `optionalAuth`
- 不使用 baseService，有独立的 messageService
- SSE 长连接 + Dialogue/Message 两张表
