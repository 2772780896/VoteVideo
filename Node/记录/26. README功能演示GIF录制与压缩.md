## 背景

README 纯文字描述项目功能缺乏直观性，插入 GIF 演示可以让 HR/面试官在 10 秒内理解项目全貌。但 ScreenToGif 直接录制的原始 GIF 体积过大（单段 20+ MB），不适合放入 GitHub 仓库，需要压缩后才能嵌入。

---

## 录制工具

使用 [ScreenToGif](https://www.screentogif.com/)（开源免费，自带帧编辑器），录制时：

- 浏览器缩放 125%-150%，确保文字清晰
- 操作前停顿 1-2 秒，让观众看清当前状态
- 每段控制在 10-25 秒，突出核心功能

## 压缩问题与方案

### 原始数据

| 文件 | 分辨率 | 帧数 | 帧间隔 | 时长 | 大小 |
|------|--------|------|--------|------|------|
| 基本浏览.gif | 2564×1532 | 246 帧 | 90ms | 22s | 21 MB |
| 用户交互.gif | 2564×1532 | 510 帧 | 190ms | 97s | 22 MB |

### 压缩策略（三个维度）

1. **降分辨率**：2564px → 800px 宽（约 3 倍缩小，像素量减少约 9 倍）
2. **抽帧**：每 N 帧取 1 帧（frame_skip=3 或 5），减少帧数
3. **减色**：256 色 → 128 色（MEDIANCUT 量化），减少每帧数据量

### 压缩脚本（Python + Pillow）

```python
from PIL import Image
import os

def compress_gif(input_path, output_path, target_width=800, frame_skip=3, duration_ms=150):
    """
    参数说明：
      target_width  - 输出宽度（等比缩放高度）
      frame_skip    - 每 N 帧取 1 帧（3 = 保留 1/3 帧）
      duration_ms   - 输出每帧间隔（毫秒）
    """
    img = Image.open(input_path)
    orig_w, orig_h = img.size
    ratio = target_width / orig_w
    target_height = int(orig_h * ratio)

    frames = []
    i = 0
    while True:
        try:
            img.seek(i)
            if i % frame_skip == 0:
                frame = img.copy().convert('RGBA')
                frame = frame.resize((target_width, target_height), Image.LANCZOS)
                # RGBA → RGB（白底）再量化为调色板模式
                bg = Image.new('RGB', frame.size, (255, 255, 255))
                bg.paste(frame, mask=frame.split()[3])
                frame = bg.quantize(colors=128, method=Image.Quantize.MEDIANCUT)
                frames.append(frame)
            i += 1
        except EOFError:
            break

    frames[0].save(
        output_path,
        save_all=True,
        append_images=frames[1:],
        optimize=True,
        duration=duration_ms,
        loop=0
    )

    orig_size = os.path.getsize(input_path) / (1024 * 1024)
    new_size = os.path.getsize(output_path) / (1024 * 1024)
    total_sec = len(frames) * duration_ms / 1000
    print(f"{target_width}x{target_height}, {len(frames)}f @ {duration_ms}ms "
          f"= {total_sec:.1f}s, {orig_size:.1f}MB -> {new_size:.1f}MB")
```

### 各文件压缩参数与结果

```python
# 基本浏览：skip=3, 128色, 150ms → 22s 压到 12s
compress_gif('基本浏览.gif', '基本浏览.gif',
             target_width=800, frame_skip=3, duration_ms=150)
# 结果：800×478, 82帧, 12.3s, 21MB → 3.1MB

# 用户交互：skip=5, 256色, 150ms → 97s 压到 15s
compress_gif('用户交互.gif', '用户交互.gif',
             target_width=800, frame_skip=5, duration_ms=150)
# 结果：800×478, 102帧, 15.3s, 22MB → 4.3MB
```

### 压缩对比汇总

| 文件 | 原始 | 压缩后 | 压缩比 |
|------|------|--------|--------|
| 基本浏览.gif | 2564×1532, 246帧, 22s, 21MB | 800×478, 82帧, 12s, 3.1MB | **85%** |
| 用户交互.gif | 2564×1532, 510帧, 97s, 22MB | 800×478, 102帧, 15s, 4.3MB | **80%** |

### 参数调优经验

- **frame_skip**：3 是流畅度和体积的平衡点；5 适合原始帧率很低（5fps）的长录制
- **colors**：128 色对 UI 类 GIF 足够（界面以纯色块为主），视频播放类建议 256 色
- **duration_ms**：150ms（约 7fps）对 UI 演示足够流畅；低于 100ms 会显著增大体积
- **target_width**：800px 是 GitHub README 的理想宽度，手机上也能完整显示

## README 嵌入方式

GIF 放在仓库 `demo/` 目录下，使用**相对路径**引用（GitHub 自动渲染）：

```markdown
## 项目预览

### 功能演示

**首页浏览与视频播放** — 轮播图、视频列表（含骨架屏加载）、详情页播放与关联推荐

![基本浏览](demo/基本浏览.gif)

**社交交互链路** — 点赞、收藏、转发、评论，未登录触发全局登录弹窗

![用户交互](demo/用户交互.gif)
```

### 注意事项

- **不要用图床**：图床可能失效，放仓库内最稳定
- **单文件上限 10MB**：GitHub 对渲染图片有大小限制，超过 10MB 不会内联显示
- **总大小控制**：多个 GIF 总计建议 < 15MB，否则 README 加载过慢
- **中文文件名**：GitHub 支持中文路径，但 `git add` 时注意终端编码（Windows cmd 下可能需要 `chcp 65001`）
