# 20. baseService.js 功能说明

## 定位

通用服务工厂 —— 传入表名和配置，返回一套现成的 CRUD 方法。避免 video/essay/post/comment/tag 五个模块各自重复写分页、排序、搜索逻辑。

---

## 工具函数（4个，独立导出）

| 函数 | 作用 |
|------|------|
| `parseSortParam` | 把 `"-date"` 转成 Prisma 能认的 `{ date: 'desc' }` |
| `parsePagination` | 把 page + element 算出 skip（跳过几条）和 take（取几条） |
| `formatDate` | Date 对象转 `"2026-06-12"` 字符串 |
| `safeJsonParse` | 安全解析 JSON，遇到纯字符串不会崩，包装成单元素数组 |

---

## createService 工厂函数

### 输入

- `modelName`：Prisma 模型名（`'video'`、`'essay'` 等）
- `config`：配置对象，包含主键字段名、搜索字段、默认排序、连表配置、转换函数、媒体类型

### 输出

返回一个对象，包含 6 个方法：

| 方法 | 作用 |
|------|------|
| `getListData` | 核心方法。分页 + 排序 + 搜索 + 按用户筛选 + 关注Feed + 连表查询 + 数据转换 + 自动合并交互状态 |
| `getItemData` | 按 ID 查单条，支持"不存在就抛404"，同样做数据转换 |
| `count` | 查满足条件的总数 |
| `create` | 创建一条记录，自动带上连表配置 |
| `update` | 按 ID 更新一条记录 |
| `delete` | 按 ID 删除一条记录 |

### 闭包机制

`createService` 执行时，配置参数（`defaultSortField`、`searchField`、`includeConfig`、`transformFunction`、`mediaType` 等）被闭包"记住"。返回的方法在后续调用时不需要再传这些配置，每个模块各自拥有独立的一套配置。

---

## getListData 详细流程

执行顺序：

```
拆参数 → 算分页 → 拼查询条件 → 确定连表配置 → 并行查数据库 → 数据转换 → 合并交互状态 → 返回
```

### 参数说明

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `page` | 1 | 第几页 |
| `element` | 16 | 每页几条 |
| `sort` | `-date` | 排序（`-`前缀表示降序） |
| `q` | `''` | 搜索关键词 |
| `where` | `{}` | 额外筛选条件 |
| `include` | `null` | 覆盖默认的连表配置 |
| `currentUid` | `null` | 当前登录用户ID |
| `uid` | — | 按用户ID筛选（用户主页） |
| `following` | — | 是否只查关注用户的内容（关注Feed） |

### 特殊能力

1. **搜索**：根据配置的 searchField 自动加模糊匹配（video 搜标题，post 搜正文）
2. **用户主页筛选**：传 `uid` 就只返回这个用户发的内容
3. **关注Feed**：传 `following=true` + `uid`，自动查关注列表，只返回关注的人发的内容
4. **连表覆盖**：默认用预设的 includeConfig，调用时可以传自定义的覆盖
5. **自动转换**：如果配了 transformFunction，每条数据自动过一遍转换函数
6. **自动合并交互**：如果传了 currentUid，自动调用 interactionService 合入 isLiked/isFavourited 等字段

---

## MODULE_CONFIG 配置表

为 5 种媒体各预设了一份配置：

| 类型 | 主键 | 搜索字段 | 默认排序 | 连表配置 |
|------|------|---------|---------|---------|
| video | vid | title | date | uploader + videoTags |
| essay | eid | title | date | uploader |
| post | pid | text | date | uploader |
| comment | cid | text | date | uploader + parentComment |
| tag | tid | tagName | tid | 无 |

各 Service 拿这份配置去 `createService`，就能拿到一套定制好的方法。

## 当前使用情况

| Service | 是否使用 baseService | 说明 |
|---------|---------------------|------|
| videoService | 是 | 列表接口直接委托给 `getListData` |
| essayService | 是 | 同上 |
| postService | 是 | 同上 |
| tagService | 是 | 同上 |
| commentService | 是 | 通过 `where` 和 `include` 覆盖复用 `getListData`，仅保留特殊的详情和创建逻辑 |
