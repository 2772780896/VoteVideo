# 11. Message接口重构总结（SSE方案）

## 日期
2026-06-12

## 背景与问题

### 初始状态
- **文件**：`controllers/messageController.js`（280行）
- **问题**：
  1. **Token验证重复**（3个函数都有，各13行，共39行）
  2. **无SSE支持**（无法实时推送消息）
  3. **无服务层**（直接操作数据库）
  4. **代码冗余**（相似逻辑重复编写）

### 核心问题：如何实现即时通信？
**问题**：单纯的HTTP GET/POST做不到即时通信，因为：
- HTTP是**请求-响应**模式，客户端必须主动请求
- 服务器无法主动推送消息给客户端

**解决方案**：**SSE (Server-Sent Events) + HTTP POST**
- SSE：服务器→客户端单向推送（实时接收消息）
- HTTP POST：客户端→服务器发送消息

---

## SSE方案设计

### 架构图
```
客户端A (发送)          服务器              客户端B (接收)
    |                   |                   |
    |--- POST 消息 ---->|                   |
    |                   |--- SSE推送 ------>|
    |                   |                   |
    |                   |<--- SSE连接 ------|
    |                   |                   |
```

### 技术选型对比

| 方案 | 技术 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| **SSE + HTTP POST** | SSE + REST | 简单，实时性好，防火墙友好 | 只能单向推送 | ⭐⭐⭐⭐⭐ |
| **WebSocket** | WebSocket | 真正的双向通信 | 实现复杂，防火墙可能拦截 | ⭐⭐⭐⭐ |
| **轮询** | HTTP GET + setInterval | 简单，兼容性好 | 延迟高，浪费资源 | ⭐⭐ |
| **长轮询** | HTTP GET + 超时 | 实时性较好 | 实现复杂，服务器压力大 | ⭐⭐⭐ |

### 为什么选SSE？
1. **比WebSocket简单**：SSE是HTTP协议，防火墙友好
2. **足够用**：聊天场景，只需要服务器→客户端推送
3. **易于实现**：前端用`EventSource`，后端用HTTP响应

### 2. 添加SSE支持（实时消息推送）

#### 新增功能：getMessageStream

```javascript
// controllers/messageController.js - getMessageStream (第8-55行)
const getMessageStream = async (req, res) => {
  try {
    const { uid } = req.user

    // 1. 设置 SSE 响应头
    res.setHeader('Content-Type', 'text/event-stream')
    res.setHeader('Cache-Control', 'no-cache')
    res.setHeader('Connection', 'keep-alive')

    // 2. 添加用户到在线列表
    messageService.addOnlineUser(uid, res)

    // 3. 发送初始消息（确认连接成功）
    res.write(`data: ${JSON.stringify({ type: 'connected', uid: uid })}\n\n`)

    // 4. 心跳（保持连接）
    const heartbeat = setInterval(() => {
      res.write(`data: ${JSON.stringify({ type: 'heartbeat' })}\n\n`)
    }, 30000)

    // 5. 连接关闭时清理
    req.on('close', () => {
      clearInterval(heartbeat)
      messageService.removeOnlineUser(uid)
      res.end()
    })

  } catch (error) {
    // ...
  }
}
```

**关键设计**：
1. **在线用户管理**：`onlineUsers` Map（uid → SSE响应对象）
2. **心跳机制**：每30秒发送一次心跳，保持连接
3. **自动清理**：连接关闭时，从在线列表移除用户

### 3. 服务层建立（提取业务逻辑）

#### 新增文件：services/messageService.js

**核心功能**：
1. **在线用户管理**：
   - `addOnlineUser(uid, res)`：添加用户到在线列表
   - `removeOnlineUser(uid)`：从在线列表移除用户
   - `isUserOnline(uid)`：检查用户是否在线
   - `pushMessageToUser(uid, message)`：推送消息给用户

2. **业务逻辑**：
   - `sendMessage(senderUid, receiverUid, text)`：发送私信
   - `getDialogues(uid)`：获取对话列表
   - `getNotifications(uid)`：获取通知列表

#### 服务层代码示例

```javascript
// services/messageService.js - sendMessage (第41-107行)
async function sendMessage(senderUid, receiverUid, text) {
  // 1. 查找或创建对话
  let dialogue = await prisma.dialogue.findFirst({...})

  if (!dialogue) {
    dialogue = await prisma.dialogue.create({...})
  } else {
    dialogue = await prisma.dialogue.update({...})
  }

  // 2. 插入消息记录
  const newMessage = await prisma.message.create({...})

  // 3. 通过SSE推送消息给接收者（如果在线）
  const messageData = {
    type: 'new_message',
    data: {...}
  }
  const pushed = pushMessageToUser(receiverUid, messageData)

  // 4. 返回结果
  return {
    messageId: newMessage.id,
    dialogueId: dialogue.mid,
    pushed: pushed
  }
}
```

**改进**：
- ✅ 控制器代码减少56%（280行 → 147行）
- ✅ 业务逻辑集中管理
- ✅ 易于测试和维护

**改进**：
- ✅ 添加SSE路由（`/stream`）
- ✅ 所有路由都使用`needToken`中间件
- ✅ 路由路径简化（去掉`/user`前缀）
