## 1. 环境安装与配置

首先切换至最高管理员权限（Root 用户），以便后续无阻碍执行系统级操作：

```bash
# 切换为 root 用户（后续命令均默认在 root 权限下执行）
sudo -i

```

更新系统并安装基础组件

```bash
# 更新本地软件源缓存
apt update && apt upgrade -y

# 安装解压工具与 Nginx Web 服务器（Nginx 会自动注册系统开机自启）
apt install unzip nginx -y

```

安装 Node.js 与 npm

由于 Ubuntu 自带仓库版本较旧，请根据实际需要通过官方或补齐方式安装（以系统自带 v18 补全为例）：

```bash
# 安装 Ubuntu 官方仓库的 nodejs
apt install nodejs -y

# 必须手动补齐配套的 npm 包管理器（勿用低版本提示引导）
apt install npm -y

# 验证安装是否成功，需正常输出版本号
node -v
npm -v

# 全局安装进程守护管理器 PM2
npm install pm2 -g

```

## 2. 项目获取与目录规范

推荐将全栈项目统一放置在 /var/www/ 规范线下，形成前后端同级目录结构：

```text
/var/www/myapp/
├── frontend/     <-- 放前端打包后的静态文件（index.html 等）
└── backend/      <-- 放后端的 Node.js 源码（app.js、package.json 等）

```

上传方案（绕过海外 Git 限速）

在本地电脑中，排除 node_modules，将前后端源码整体压缩为 project.zip，通过 SFTP/拖拽上传至服务器 /var/www/：

```bash
# 进入 Web 服务器默认根目录
cd /var/www/

# 解压上传的压缩包到指定项目文件夹 myapp
unzip project.zip -d myapp

```

## 3. 后端部署 (Node.js + PM2)

```bash
# 1. 精准进入后端子目录
cd /var/www/myapp/backend

# 2. 安装生产环境依赖（忽略开发工具，节省空间与时间）
npm install --production

# 3. 使用 PM2 在后台永久启动后端服务（假设入口为 app.js）
pm2 start app.js --name "vote-backend"

# 4. 配置 PM2 自身开机自启（执行后请根据屏幕提示复制并运行超长指令）
pm2 startup

# 5. 保存当前运行的进程快照，确保服务器重启后自动恢复服务
pm2 save

```

## 4. 前端部署 (React 静态托管)

React 作为单页应用（SPA），严禁在服务器运行 npm start。

方案 A（本地打包，推荐）：在本地电脑运行 npm run build，将生成的 build/ 或 dist/ 文件夹通过 SFTP 上传至服务器 /var/www/myapp/frontend/ 目录下。

方案 B（服务器现场打包）：

```bash
cd /var/www/myapp/frontend
npm install
npm run build  # 现场编译生成 build 文件夹

```

## 5. Nginx 反向代理配置

通过配置 Nginx，实现外网通过 80 端口访问时，前端直接走静态托管，后端请求（/api）转发给内网的 Node.js 端口。

```bash
# 编辑 Nginx 默认配置文件
nano /etc/nginx/sites-enabled/default

```

将文件内容修改/替换为以下核心配置（注：确保 proxy_pass 端口与 Node.js 监听端口一致）：

```nginx
server {
    listen 80 default_server;
    listen [::]:80 default_server;

    server_name _; # 绑定服务器公网 IP

    # 1. 静态托管 React 前端
    location / {
        root /var/www/myapp/frontend/build; # 精准指向包含 index.html 的打包目录
        index index.html index.htm;
        try_files $uri $uri/ /index.html;   # 解决 React-Router 刷新报 404 的核心配置
    }

    # 2. 反向代理 Node.js 后端接口
    location /api/ {
        proxy_pass http://127.0.0.1:3000;   # 转发给内网 Node.js 端口
        
        # 标准反向代理请求头透传
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        
        # 传递客户端真实 IP
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

```

生效配置

```bash
# 检查 Nginx 配置文件语法是否有误
nginx -t

# 语法无误后，平滑重启 Nginx 使配置生效
nginx -s reload

```

⚠️ 注意：请前往腾讯云控制台的安全组/防火墙中，开放 80 端口。无需开放后端的 3000 端口。

## 6. 常用维护命令

### PM2 (后端管理)

```bash
pm2 list
pm2 log
pm2 restart vote-backend
pm2 stop vote-backend
```

- `pm2 list`：查看所有后台运行的服务状态。
- `pm2 log`：查看实时的后端报错与运行日志。
- `pm2 restart vote-backend`：重启后端服务。
- `pm2 stop vote-backend`：停止后端服务。

### Nginx (服务管理)

```bash
systemctl status nginx
systemctl restart nginx
```

- `systemctl status nginx`：查看 Nginx 运行状态。
- `systemctl restart nginx`：强制重启 Nginx 服务。
