1. Prisma Schema / 多态设计（最容易补、面试必问）

你项目里的"多态"其实非常简单，就是一张表管多种内容类型。具体来说：如果不用多态，你需要建 VideoLike、EssayLike、PostLike 三张表；你现在的做法是用一张 UserLike 表，通过 type 字段区分（'video'/'essay'/'post'），item_id 指向对应内容的主键。

这个设计的好处是新加一种内容类型（比如"音频"）不需要建新表，只需要在 INTERACTION_CONFIG 里加一条配置。代价是不能用 Prisma 的 include 做关联查询（因为 item_id 可能是视频ID也可能是文章ID，数据库不知道关联哪张表），所以你写了 interactionService.checkInteractions() 来手动批量查询再合并。
