# 五媒体后端流程对比（以 Video 为基准）

> 本文档以 `22. Video后端全流程.md` 为基准，只记录各媒体与 Video 的**差异点**。
> 相同部分（optionalAuth、baseService 通用方法、transformer 模式、mergeInteractions、response 工具）不再重复。

---

## 总览：API 矩阵

| 媒体 | 列表 GET / | 详情 GET /:id | 推荐 GET /related |
|------|-----------|---------------|-------------------|
| video | ✓ | ✓ `:vid` | ✓（**tag 匹配**） |
| essay | ✓ | ✓ `:eid` | ✓（取最新） |
| post | ✓ | ✓ `:pid` | ✓（取最新） |
| tag | ✓（默认 `-likeCount`） | ✓ `:tid` | ✓（取最热） |
| comment | ✓（**多资源过滤**） | ✓ `:cid` | — |

> 创建类接口统一由 `uploadService` 处理（`POST /api/upload/video`、`/essay`、`/post`），不放在各媒体的 controller 中。

---

## Essay — 与 Video 的差异

**路由**：无 `/main` 别名，其余结构相同（`/` → 列表，`/related` → 推荐，`/:eid` → 详情）

**Service 差异**：
- `createService` 配置：`idField: 'eid'`，`searchField: 'title'`，无 `tagRelationField`
- 推荐走 `baseService.getRelatedData` 的**无 tag 回退路径**（排除自己，取最新）
- transformer 无 `videoUrl`、无 `videoTags`、无 `duration`，多出 `text`（文章正文）

**其余完全相同**：optionalAuth、baseService.getListData、mergeInteractions、sendList/sendSuccess

---

## Post — 与 Video 的差异

**路由**：同 essay，无 `/main` 别名

**Service 差异**：
- `idField: 'pid'`，无 `tagRelationField`
- 推荐同 essay（无 tag 回退路径）
- transformer 输出 `text`、`pictureList`（JSON 数组，用 safeJsonParse 解析）、`videoList`

---

## Tag — 与 Video 的差异

**Service 差异**：
- `idField: 'tid'`，`searchField: 'tagName'`，无 `tagRelationField`
- **默认排序不同**：`-likeCount`（热度），不是 `-date`。热门标签通过列表接口 `GET /api/tag?sort=-likeCount&element=10` 实现
- `includeConfig` 无 uploader（标签没有上传者）
- transformer 输出 `tid, tagName, likeCount, favouriteCount, commentCount`
- 推荐同无 tag 回退路径，但默认按 `-likeCount` 排序

---

## Comment — 与 Video 的差异最大

**路由**：只有 `GET /`（列表）和 `GET /:cid`（详情），**无推荐接口**

**列表接口的特殊之处**：
- 接受 `vid/eid/pid/tid` 四个过滤参数（评论属于特定资源）
- **排序参数格式不同**：前端传 `1`（热度降序）、`-1`（热度升序）、`date`、`-date`
- Controller 里有一段参数转换逻辑（`'1'` → `'-likeCount'`，`'-1'` → `'likeCount'`）
- Service 用 `baseService.getListData` 的 `where` + `include` 覆盖：
  - `where: { replyTo_cid: null }`（只取顶级评论）
  - `include: COMMENT_INCLUDE`（包含 replyCount 子查询）

**Service 差异**：
- `createService` 配置：`idField: 'cid'`，无搜索字段
- transformer 输出 `cid, vid, text, type, pictureList, likeCount, commentCount, date`
- 用 `safeJsonParse` 解析 `pictureList`（数据库存 JSON 字符串）

---

## Message — 完全不同的体系（建议单独文档）

**认证方式**：`needToken`（强制认证），不是 `optionalAuth`

**接口列表**：
| 接口 | 方法 | 说明 |
|------|------|------|
| `/stream` | GET | SSE（Server-Sent Events）实时消息流 |
| `/send` | POST | 发送私信（需 `receiverUid, text`） |
| `/dialogues` | GET | 获取对话列表 |
| `/notifications` | GET | 获取通知列表 |

**Service 差异**：
- **不使用 baseService**，有自己的 messageService
- SSE 长连接：保持 HTTP 连接，服务端推送消息
- 涉及 `Dialogue`（对话）和 `Message`（消息）两张表
- 与内容媒体（video/essay/post/tag/comment）完全独立
