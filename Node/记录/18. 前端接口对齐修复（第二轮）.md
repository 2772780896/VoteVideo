# VoteVideo 后端修复记录 — 前端接口对齐（第二轮）

日期：2026-06-14
基准文档：`13. VoteVideo-前端接口分析.md`（前端 22 个 API 接口完整分析）

---

## 问题背景

基于前端接口文档（13号文件）对后端 Node 代码进行逐项遍历对比，发现上一轮修复（15/16/17号文件）后仍存在以下问题：

1. **缺失接口**：历史记录（#22）和回复（#20）两个前端接口在后端完全没有实现
2. **详情页交互状态始终为 false**：所有媒体类型的详情接口（视频/文章/图文/评论/标签）的 `isLiked`、`isFavourited`、`isReshared` 等字段始终返回 `false`
3. **列表交互查询不完整**：视频/文章列表缺少 `isReshared`，图文列表缺少 `isFavourited` 和 `isReshared`
4. **关注动态 Feed 不可用**：列表接口不支持 `following=true` 查询参数
5. **评论功能缺陷**：评论列表只支持按 `vid` 筛选，不支持 `eid`/`pid`/`tid`；缺少 `parentType`/`parentId` 字段
6. **标签交互缺失**：标签详情不返回任何交互状态字段
7. **RESOURCE_CONFIG 不完整**：评论和标签不支持收藏（`hasFavourite: false`），与前端期望不符

## 详细变更说明

### 1. 数据库模型变更 — `prisma/schema.prisma`

**Comment 模型新增字段：**
- `tid Int?` — 标签ID（可选），用于支持标签下的评论
- `tag Tag?` — 关联到 Tag 模型

**Tag 模型新增关联：**
- `comments Comment[]` — 标签下的评论（反向关联）

**部署注意：** 需要运行 `npx prisma db push` 同步数据库结构。

---

### 2. 新增接口：历史记录 — `POST /api/{mediaType}/{mediaId}/history`

**前端文档 #22：** 用户查看视频/文章/图文时自动调用，静默失败。

**interactService.js — `recordHistory(uid, type, id)`：**
- 检查资源是否存在
- 如果 `hasHistory: false`（评论、标签），静默返回成功
- 如果已有历史记录，先删除再创建（更新 addDate 到最新）
- 创建 UserHistory 记录

**interactController.js — `history`：**
- 从 `req.params` 提取 `type` 和 `id`
- 调用 `recordHistory`，返回 `{ code: 200, message: '记录成功' }`

**interactRoutes.js：**
- `POST /:type/:id/history` — 挂载 `needToken`

---

### 3. 新增接口：回复 — `POST /api/{mediaType}/{mediaId}/reply`

**前端文档 #20：** 在所有媒体类型下创建评论，请求体 `{ text: "回复内容" }`。

**interactService.js — `createReply(uid, mediaType, mediaId, text)`：**
- 参数校验：text 不能为空
- 检查资源存在且支持回复
- 根据 mediaType 设置评论的父级关联字段：
  - `video` → `vid`
  - `essay` → `eid`
  - `post` → `pid`
  - `tag` → `tid`
  - `comment` → `replyTo_cid` + 继承父评论的资源关联字段
- 回复评论时增加父评论的 `subCommentCount`
- 回复其他资源时增加该资源的 `commentCount`

**interactController.js — `reply`：**
- 返回 `{ code: 201, message: '回复成功', data: { cid } }`

**interactRoutes.js：**
- `POST /:type/:id/reply` — 挂载 `needToken`

---

### 4. RESOURCE_CONFIG 扩展 — `interactService.js`

每个资源类型新增 `hasHistory` 和 `hasReply` 配置项，并调整了 `hasFavourite`/`hasDislike` 以匹配前端期望：

| 类型 | hasLike | hasFavourite | hasReshare | hasDislike | hasHistory | hasReply |
|------|---------|-------------|-----------|-----------|-----------|---------|
| video | ✅ | ✅ | ✅ | ❌ | ✅ | ✅ |
| essay | ✅ | ✅ | ✅ | ❌ | ✅ | ✅ |
| post | ✅ | ✅ | ✅ | ❌ | ✅ | ✅ |
| comment | ✅ | ✅ (改) | ❌ | ✅ | ❌ | ✅ |
| tag | ✅ | ✅ (改) | ❌ | ✅ (改) | ❌ | ✅ |

**变更说明：**
- comment: `hasFavourite` 从 `false` 改为 `true`（前端支持收藏评论）
- tag: `hasFavourite` 从 `false` 改为 `true`（前端支持收藏标签）
- tag: `hasDislike` 从 `false` 改为 `true`（前端支持踩标签）

### 5. 详情页交互状态修复

**问题：** 所有详情页的 `isLiked`/`isFavourited`/`isReshared` 始终返回 `false`。

**根因：** `getXxxDetailData` 函数调用 `transformXxxData` 时没有传入 `currentUid`，且交互状态查询不完整。

**修复：**

| 文件 | 变更 |
|------|------|
| `videoService.getVideoDetailData` | 传入 `currentUid` 到 transform + 应用 `isReshared` |
| `essayService.getEssayDetailData` | 传入 `currentUid` 到 transform + 应用 `isReshared` |
| `postService.getPostDetailData` | 传入 `currentUid` 到 transform + 应用 `isFavourited` + `isReshared` |
| `commentService.getCommentDetailData` | 新增 `currentUid` 参数 + 查询 `isLiked`/`isDisliked`/`isFavourited` |
| `tagService.getTagDetailData` | 新增 `currentUid` 参数 + 查询 `isLiked`/`isDisliked`/`isFavourited` |

---

### 6. 列表交互查询补全

**问题：** 列表接口的交互状态查询缺少 `isReshared`（video/essay/post）和 `isFavourited`（post）。

**修复：**

| 函数 | 新增查询 |
|------|---------|
| `checkVideoInteractions` | 新增 UserReshare 查询 → 返回 `isReshared` |
| `checkEssayInteractions` | 新增 UserReshare 查询 → 返回 `isReshared` |
| `checkPostInteractions` | 新增 UserFavourite + UserReshare 查询 → 返回 `isFavourited` + `isReshared` |

控制器层面同步更新：`videoController`、`essayController`、`postController` 的列表和相关推荐接口都应用了完整的 `isLiked`/`isFavourited`/`isReshared` 三个状态。

---

### 7. 关注用户动态 Feed — `following` 参数

**前端文档 #12：** `GET /api/{resourceType}?uid={uid}&following=true` 获取关注用户的内容。

**baseService.js 修改：**
在 `getListData` 中新增 `following` 参数处理：
1. 当 `following=true` 且 `uid` 有值时，查询 `UserFollowing` 表获取关注列表
2. 如果关注列表为空，直接返回 `{ data: [], total: 0 }`
3. 否则用关注列表的 uid 列表作为 `uploader_uid` 的 `in` 条件

**commentService.js 同步实现：** 在 `getCommentListData` 中也新增了相同的 `following` 逻辑。

**控制器更新：** `videoController`、`essayController`、`postController`、`commentController` 都从 `req.query` 提取 `following` 参数并传入服务。

---

### 8. 评论服务全面修复 — `commentService.js`

**问题汇总：**
- 只支持 `vid` 筛选，不支持 `eid`/`pid`/`tid`
- 缺少 `parentType`/`parentId` 字段
- 缺少 `isFavourited`/`isReshared` 字段
- 列表不查询交互状态

**修复内容：**

`transformCommentData` 新增字段：
- `parentType` / `parentId` — 根据 `vid`/`eid`/`pid`/`tid` 自动推断
- `isFavourited` — 评论收藏状态
- `isReshared` — 固定 `false`（评论不支持转发）
- `reshareCount` — 固定 `'0'`

`getCommentListData` 新增参数：
- `eid`、`pid`、`tid` — 按父级资源类型筛选
- `following` + `uid` — 关注用户动态筛选
- `currentUid` — 交互状态查询（`isLiked`/`isDisliked`/`isFavourited`）

`getCommentDetailData` 新增：
- `currentUid` 参数
- 查询 `UserLike`/`UserDislike`/`UserFavourite` 获取完整交互状态

---

### 9. 标签服务修复 — `tagService.js`

**问题：** 标签详情不返回任何交互状态字段，前端 InteractionBar 无法正常工作。

**修复：**

`transformTagData` 新增字段：
- `isLiked`、`isDisliked`、`isFavourited`、`isReshared` — 默认 false
- `reshareCount` — 固定 `'0'`

`getTagDetailData` 新增：
- `currentUid` 参数
- 查询 `UserLike`/`UserDislike`/`UserFavourite` 获取交互状态

`tagController.getTagDetail` 新增：
- 从 `req.user` 提取 `currentUid` 并传入服务
