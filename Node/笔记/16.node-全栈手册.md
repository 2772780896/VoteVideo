> 本笔记完美整合了：Node 基础、模块化、核心内置模块、npm 包管理、Webpack 全套配置、以及 Express 后端实战进阶。

## 一、 Node.js 基础与模块化

### 1. 运行环境
- **Node.js**：JS 的后端运行环境，脱离浏览器运行。
- **限制**：无法使用 DOM 和 BOM。
- **运行**：终端输入 `node xxx.js`。

### 2. 导入导出 (模块化)
#### CommonJS (CJS) - Node 默认
- **导出**：`module.exports = { name: 'zs' }`
- **导入**：`const m = require('./xxx.js')`
#### ECMAScript (ESM) - 现代标准
- **配置**：`package.json` 写入 `{"type": "module"}`。
- **导出**：`export default { ... }` 或 `export const a = 1`。
- **导入**：`import m from './xxx.js'` 或 `import { a } from './xxx.js'`。

## 二、 核心内置模块 (详解与实战)

### 1. path 模块：路径处理
- **__dirname**：当前文件的父文件夹绝对路径。
- **path.join()**：跨平台拼接路径（自动处理斜杠）。
  ```javascript
  const path = require('path');
  const fullPath = path.join(__dirname, 'data', '1.txt');
  ```

### 2. fs 模块：文件读写
- **同步/异步写法**：建议使用 `fs.promises` 以支持 `async/await`。
  ```javascript
  const fs = require('fs').promises;
  // 写入
  await fs.writeFile('./test.txt', '内容', 'utf-8');
  // 读取
  const data = await fs.readFile('./test.txt', 'utf-8'); 
  // 注意：如果不加 'utf-8'，读出来的是 Buffer (十六进制)，需 data.toString() 转换。
  ```

### 3. http 模块：创建 Web 服务 (了解原理)
```javascript
const http = require('http');
const server = http.createServer((req, res) => {
  // 设置响应头，防止中文乱码
  res.setHeader('Content-Type', 'text/html; charset=utf-8');
  if (req.url === '/index') {
    res.end('首页');
  }
});
server.listen(3000, () => console.log('启动在 3000'));
```

## 三、 npm 部署

### 1. 基础命令
- `npm init -y`：初始化清单文件 `package.json`。
- `npm i 包名`：下载包，记录到 `dependencies`。
- `npm i`：根据清单自动重装所有包。
- `npm uni 包名`：删除包。

### 2. nodemon
- **作用**：代码保存后自动重启服务。
- **安装**：`npm i -g nodemon`。

### 3.Node项目目录结构
```

├── app.js                  # 入口：创建服务器、挂载中间件、启动监听

    └── ...
```

## 四、 Webpack 构建工具 (详细配置)

> **Webpack 核心五要素**：Entry, Output, Loader, Plugin, Mode。

### 1. 基础出入口
```javascript
module.exports = {
  entry: './src/main.js',
  output: {
    path: path.join(__dirname, 'dist'),
    filename: 'bundle.js',
    clean: true // 打包前清空 dist
  }
}
```

### 2. 常用加载器 (Loader) - 翻译官
- **CSS 处理**：
  - `css-loader`：让 JS 能识别 CSS。
  - `style-loader`：将 CSS 注入 HTML。
- **图片处理 (Asset)**：
  - `type: 'asset'`：自动判断（小于 8kb 转 base64，大于则生成文件）。
  ```javascript
  generator: { filename: 'assets/[hash][ext]' } // 设置导出路径和随机名
  ```

### 3. 常用插件 (Plugin) - 功能增强
- **html-webpack-plugin**：自动生成 HTML 并引入 JS。
- **mini-css-extract-plugin**：提取 CSS 为独立文件。
- **css-minimizer-webpack-plugin**：压缩 CSS 体积。

### 4. 开发服务器 (webpack-dev-server)
- **作用**：实时重新加载。
- **配置**：`devServer: { static: './dist' }`。
- **启动**：`npm run dev` (需在 scripts 配置 `"dev": "webpack serve --open"`)。

## 六、 进阶实战：你还需要学什么？

### 1. 数据库交互 (基础：MySQL 原生连接)
- **核心包**：`npm i mysql2`。
- **为什么要用连接池？**：频繁创建和销毁连接非常耗性能。连接池可以复用连接，提高效率。
- **实战配置 (db.js)**：
  ```javascript
  const mysql = require('mysql2/promise');

  // 创建连接池
  const pool = mysql.createPool({
    host: 'localhost',
    user: 'root',
    password: '密码',
    database: '数据库名',
    waitForConnections: true,
    connectionLimit: 10 // 最大连接数
  });

  module.exports = pool;
  ```
- **在 Express 中使用**：
  ```javascript
  const db = require('./db');

  app.get('/users', async (req, res) => {
    try {
      // 使用 execute 防止 SQL 注入
      const [rows] = await db.execute('SELECT id, username FROM users WHERE age > ?', [18]);
      res.send({ code: 200, data: rows });
    } catch (err) {
      res.status(500).send('服务器错误');
    }
  });
  ```

### 2. 数据库进阶：Prisma (ORM)
- **Node.js、Express 与 Prisma 的关系**：
  - **Node.js**：运行环境（地基）。
  - **Prisma**：数据库操作工具（挖掘机）。它运行在 Node.js 环境中，不依赖于 Express，但常被用在 Express 的路由函数里。
  - **Express**：Web 框架（房子）。它负责接收前端请求，然后调用 Prisma 去数据库拿数据。

- **Prisma 的核心优势**：
  - **强类型**：配合 TypeScript，写代码时能自动提示数据库所有字段名。
  - **模型驱动**：你只需要定义一个 `schema.prisma` 文件，它会自动帮你建表。

- **Prisma 快速上手流程**：
  1. **初始化**：`npx prisma init` (生成 schema 文件)。
  2. **定义模型**：
     ```prisma
     // schema.prisma
     model User {
       id    Int     @id @default(autoincrement()) // @id 主键，@default(autoincrement()) 自增
       name  String
       email String  @unique // @unique 唯一索引
       posts Post[]  // 关联模型
     }
     ```
  3. **同步数据库**：`npx prisma db push`。
  4. **CRUD 核心语法**：
     - **增**：`prisma.user.create({ data: { name: 'zs', email: 'zs@qq.com' } })`
     - **删**：`prisma.user.delete({ where: { id: 1 } })`
     - **改**：`prisma.user.update({ where: { id: 1 }, data: { name: 'ls' } })`
     - **查 (所有)**：`prisma.user.findMany()`
     - **查 (条件)**：`prisma.user.findMany({ where: { name: { contains: 'z' } } })` // 包含 'z' 的
     - **查 (排序/分页)**：`prisma.user.findMany({ skip: 0, take: 10, orderBy: { id: 'desc' } })`

### 3. 身份鉴权 (基础：Session 模式)
- **核心包**：`npm i express-session`。
- **原理**：用户登录后，服务器在内存中开辟一块空间存储用户信息，并给浏览器发一个 `connect.sid` 的 Cookie。浏览器下次请求带上 Cookie，服务器就能认出用户。
- **实战配置**：
  ```javascript
  const session = require('express-session');

  app.use(session({
    secret: 'itheima', // 密钥，用于加密 Cookie
    resave: false,
    saveUninitialized: true
  }));

  // 1. 登录时存数据
  app.post('/api/login', (req, res) => {
    if (req.body.username === 'admin' && req.body.password === '123') {
      req.session.user = req.body; // 存入 Session
      req.session.isLogin = true;
      res.send('登录成功');
    }
  });

  // 2. 取数据 (判断是否登录)
  app.get('/api/username', (req, res) => {
    if (!req.session.isLogin) return res.status(401).send('未登录');
    res.send({ username: req.session.user.username });
  });
  ```
- **局限性**：无法跨域使用，且服务器重启后 Session 会丢失。

### 4. 身份鉴权 (进阶：JWT 模式)
- **核心包**：`npm i jsonwebtoken`。
- **JWT 原理 (三段式)**：
  1. **Header**：算法信息。
  2. **Payload**：用户信息（Base64 编码，不可放敏感数据）。
  3. **Signature**：签名（Header+Payload+密钥加密），防止篡改。
- **JWT 交互流程**：
  1. 登录成功 -> 后端 `jwt.sign()` 发放 Token。
  2. 前端存储 Token -> 之后请求放入 Header 的 `Authorization` 字段。
  3. 后端拦截器 -> `jwt.verify()` 校验密钥。
- **实战流程**：
  1. **登录成功签发 Token**：
     ```javascript
     const jwt = require('jsonwebtoken');
     const secretKey = 'itheima666';

     app.post('/api/login', (req, res) => {
       // ... 校验成功后
       const user = { id: 1, username: 'admin' };
       const token = jwt.sign(user, secretKey, { expiresIn: '10h' });
       res.send({ status: 200, token: 'Bearer ' + token });
     });
     ```
  2. **验证 Token (中间件模式)**：
     ```javascript
     const authMiddleware = (req, res, next) => {
       const token = req.headers['authorization']?.split(' ')[1];
       if (!token) return res.status(401).send('请登录');

       jwt.verify(token, secretKey, (err, decoded) => {
         if (err) return res.status(401).send('无效 Token');
         req.user = decoded; // 解析出的用户信息挂载到 req
         next();
       });
     };
     ```

### 5. RESTful API 设计规范
- **核心思想**：URL 代表资源，HTTP Method 代表动作。
- **实战标准**：
  - **GET `/users`**：获取用户列表。
  - **GET `/users/:id`**：获取指定 ID 的用户信息。
  - **POST `/users`**：新增一个用户（数据在请求体 `req.body`）。
  - **PUT `/users/:id`**：修改指定 ID 的用户信息（全量更新）。
  - **DELETE `/users/:id`**：删除指定 ID 的用户。
- **状态码规范**：
  - `200 OK`：操作成功。
  - `201 Created`：新增成功。
  - `400 Bad Request`：前端传参有误。
  - `401 Unauthorized`：未登录/Token无效。
  - `403 Forbidden`：没权限访问。
  - `404 Not Found`：资源不存在。
  - `500 Internal Server Error`：后端代码崩溃。

### 6. 环境变量与安全 (dotenv)
- **为什么需要它？**：
  - **安全性**：不要把数据库密码、JWT 密钥直接写在代码里，否则上传到 GitHub 所有人都能看到。
  - **工作原理**：`.env` 是个普通文本，Node 默认不认识。`dotenv` 模块负责读取该文件并将其内容注入到 `process.env` 内存对象中。
- **实战流程**：
  1. **安装**：`npm i dotenv`。
  2. **创建文件**：在项目根目录新建 `.env` 文件（**注意：要在 .gitignore 中忽略它**）。
     ```text
     PORT=3000
     DB_PASSWORD=root123456
     ```
  3. **在入口加载**：在 `app.js` 的第一行写上：`require('dotenv').config();`
  4. **代码中使用**：通过 `process.env.变量名` 访问。

- **协作与部署方案**：
  - **GitHub 协作**：创建一个 **`.env.example`** 文件上传，只写键名（如 `DB_PASS=`），不写具体值。别人下载后改名为 `.env` 填入自己的密码。
  - **线上部署**：线上环境通常不上传 `.env` 文件，而是直接在云平台的后台管理页面设置“环境变量”，Node 依然能通过 `process.env` 拿到。
