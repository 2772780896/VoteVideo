### 已完成工作（续）

#### 4. 修复 sort=-date 导致的 500 错误
- **问题根因**：前端传 `sort=-date`，但 Prisma Schema 中字段名为 `createdAt`，导致 Prisma 查询报错
- **解决方案**：统一使用 `date` 字段，放弃 `createdAt`
- 修改 `schema.prisma`：所有模型的 `createdAt DateTime @default(now())` → `date DateTime @default(now())`
- 修改 `baseService.js`：`parseSortParam` 默认值和 `MODULE_CONFIG` 中的 `defaultSortField` 全部改为 `'date'`
- 修改所有 Service 文件：`formatDate(xxx.createdAt)` → `formatDate(xxx.date)`
- 修改所有 Controller 文件：`createdAt: true`（Prisma select）改为 `date: true`，排序条件改为 `date`
- 修改 `videoService.js` `getRelatedVideosData` 默认排序参数：`'-createdAt'` → `'-date'`
- 修改 `commentService.js` 默认排序参数：`'-createdAt'` → `'-date'`

#### 5. 修复 React Router basename 与 Vite base 不一致问题
- 问题：`Router basename="/VoteVideo"` 与访问路径 `/` 不匹配，导致路由无法渲染
- 修复：`React/src/router/index.jsx` 中 `basename` 改为 `'/'`（与 `vite.config.js` 的 `base: '/'` 一致）

### 技术细节

#### 前端 API 路径定义
- 认证：`/api/login`, `/api/register`
- 用户：`/api/user/profile`, `/api/user/:uid`, `/api/user/profile/:profileType/:dataType`
- 内容：`/api/video/:id`, `/api/video`, `/api/video/main`, `/api/video/related`
- 交互：`/api/:mediaType/:mediaId/like`, `/api/:mediaType/:mediaId/favourite` 等
- 消息：`/api/user/message/send`

#### 前端期望的数据格式
- 列表接口：`{ code: 200, data: { items: [...], total: number } }`
- 详情接口：`{ code: 200, data: {...} }`
- 操作接口：`{ code: 200, message: '操作成功', data: {...} }`
