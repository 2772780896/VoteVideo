# Zustand 使用指南

## 一、基础概念

### 1. 什么是 Zustand
Zustand 是一个轻量级的 React 状态管理库，不需要 Provider 包裹，API 简洁。

### 2. 核心 API

#### create() - 创建 store
```js
import { create } from 'zustand'

const useStore = create((set, get) => ({
  // 状态
  count: 0,
  
  // 方法
  increment: () => set(state => ({ count: state.count + 1 })),
}))
```

#### set() - 更新状态
```js
// 方式一：直接传对象（合并更新）
set({ count: 1 })

// 方式二：传入函数（基于旧状态更新）
set(state => ({ count: state.count + 1 }))

// 方式三：完全替换（第二个参数为 true）
set({ count: 1 }, true)
```

#### get() - 获取状态（在 store 内部使用）
```js
const useStore = create((set, get) => ({
  count: 0,
  doubleCount: () => get().count * 2,  // 在 store 内部获取状态
}))
```

## 二、在组件中使用

### 1. 基本用法（推荐：使用 selector）

```js
import useAuthStore from '@/stores/authStore'

const MyComponent = () => {
  // ✅ 推荐：使用 selector，只有订阅的状态变化时才重渲染
  const isLoggedIn = useAuthStore(state => state.isLoggedIn)
  const login = useAuthStore(state => state.login)
  
  return <div>...</div>
}
```

### 2. 不推荐用法（解构）

```js
// ❌ 不推荐：解构会失去响应性，且任何状态变化都会导致重渲染
const { isLoggedIn, login } = useAuthStore()
```

### 3. 在组件外使用

```js
// 在工具函数、拦截器等地方使用
import useAuthStore from '@/stores/authStore'

const token = useAuthStore.getState().user?.token
useAuthStore.getState().logout()
```

## 三、性能优化

### 1. 使用 selector 避免不必要的重渲染

```js
// ✅ 好的做法：只订阅需要的状态
const isLoggedIn = useAuthStore(state => state.isLoggedIn)

// ❌ 不好的做法：订阅整个 store
const store = useAuthStore()
```

### 2. 复杂状态的 selector

```js
// 如果状态是对象，需要使用浅比较
import { useShallow } from 'zustand/react/shallow'

const { isLoggedIn, user } = useAuthStore(
  useShallow(state => ({ isLoggedIn: state.isLoggedIn, user: state.user }))
)
```
