## 二、低代码 AI 编排平台架构解构

通过解构 Flowise / Dify，看清可视化画布背后的底层。

### 2.1 核心铁三角

| 组件 | 角色 | 职责 |
|------|------|------|
| ReactFlow | 前端棋盘 | 纯 DOM 状态管理，绝对定位 `<div>` 渲染节点，SVG 渲染连线 |
| LangChain | 后端胶水 | AI 统一接口框架，抹平各家大模型 API 差异，内置标准 AI 功能积木 |
| JSON 协议 | 通用语言 | 图形与代码的桥梁，用于存档、跨语言通信、运行时监控断点 |

### 2.2 画布核心事件监听

Flowise 前端核心组件 `packages/ui/src/views/canvas/index.tsx` 中，reactflow 钩子触发的三大核心行为：

| 事件 | 触发时机 | 本质 |
|------|----------|------|
| `onNodesChange` | 拖拽/选中卡片时 | 盲调工具库算坐标，实时同步到 Zustand 全局状态机 |
| `onConnect` | 松开连线时 | 通过 JSON 绑定变量赋值关系，修改目标节点的 inputs 数据结构 |
| `saveFlow` | 点击保存时 | `JSON.stringify(nodes + edges)` 发给后端存档 |

## 三、自然语言驱动布局修改 — 端云混合架构

**痛点**：全量 HTML 喂给 AI 导致上下文爆炸、Token 极贵、大模型首字延迟高；端侧小模型智商不足。

**端云混合（Hybrid）动态切片方案**：

```
[用户口述指令] "把右边那个表格删了"
    │
[端侧小模型 (本地 Phi-3 / Transformers.js)] → 0 延迟、0 成本语义识别
    │
吐出目标组件 ID: "comp_1"
    │
[前端 JS 拦截] → 精准切下 comp_1 的轻量 JSON 片段
    │
只发送局部上下文
    │
[云端大模型 (Dify API / 样式 Agent)] → 秒级吐出新 Tailwind JSON
    │
[前端局部热更新]
```

## 四、Component Mapper 动态渲染架构

放弃"让 AI 直接重写 React 代码并浏览器端编译"的高风险方案，采用 JSON 协议驱动 + Tailwind CSS 盲填。

### 4.1 组件原子化与语义化打标

```jsx
// 编写盲盒组件，赋予简短 description 供 AI 识别
function MyButton({ className, text }) {
    return <button className={`px-4 py-2 rounded ${className}`}>{text}</button>;
}
```

### 4.2 建立映射字典

```jsx
// 大路由字典：JSON type → React 组件
const ComponentMapper = {
    "button": MyButton,
    "input": MyInput,
    "grid": MyGridContainer
};
```

### 4.3 循环动态渲染

```jsx
function LayoutRender({ jsonLayout }) {
    return (
        <div className="layout-container">
            {jsonLayout.map((item) => {
                const Component = ComponentMapper[item.type];
                return <Component key={item.id} {...item.props} />;
            })}
        </div>
    );
}
```
