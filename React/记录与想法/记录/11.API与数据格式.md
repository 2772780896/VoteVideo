## 一、通用基础

### 1.1 axios 实例（src/utils/request.js）

```js
const request = axios.create({ baseURL: '', timeout: 5000 })
```

**请求拦截器（needToken 机制）：**
- `config.needToken === true` 时，从 `Cookies.get('token')` 取 token
- 非 DELETE 请求：改 method 为 POST，`config.data = { ...原data, authorization: token }`
- DELETE 请求：保持 method 不变，`config.data = { ...原data, authorization: token }`
- 合并而非替换 data，防止覆盖上传等请求的原有数据

**响应拦截器：**
- `response.data.code !== 200` → `Promise.reject(response)`
- `code === 200` → resolve 整个 response 对象

### 1.2 通用响应格式

```ts
{
  code: 200,          // 200=成功, 401=未认证, 404=未找到
  message: 'ok',
  data: T | T[],
  total?: number      // 列表接口附加，表示总条数
}
```

### 1.3 通用请求函数（src/apis/content.js）

**fetchItem — 获取单个资源：**

```js
const fetchItem = (resourceType, id) => request.get(`/api/${resourceType}/${id}`)
```

| 调用示例 | 实际请求 |
|---------|---------|
| `fetchItem('video', 123)` | `GET /api/video/123` |
| `fetchItem('essay', 45)` | `GET /api/essay/45` |
| `fetchItem('post', 78)` | `GET /api/post/78` |
| `fetchItem('user', 1)` | `GET /api/user/1` |
| `fetchItem('tag', 5)` | `GET /api/tag/5` |

**fetchItemList — 获取资源列表：**

```js
const SUBTYPE_MAP = {
  video:   { main: '/api/video/main', related: '/api/video/related' },
  picture: { carousel: '/api/picture/carousel' },
}

const fetchItemList = (resourceType, params = {}) => {
  const { subType, ...rest } = params
  const baseUrl = SUBTYPE_MAP[resourceType]?.[subType] ?? `/api/${resourceType}`
  return request.get(baseUrl, { params: rest })
}
```

| 调用示例 | 实际请求 |
|---------|---------|
| `fetchItemList('video', { sort, page, element })` | `GET /api/video?sort=..&page=..&element=..` |
| `fetchItemList('video', { subType: 'main', page, element })` | `GET /api/video/main?page=..&element=..` |
| `fetchItemList('video', { subType: 'related', vid, sort, page, element })` | `GET /api/video/related?vid=..&sort=..` |
| `fetchItemList('comment', { sort, page, element, vid })` | `GET /api/comment?sort=..&page=..&vid=..` |
| `fetchItemList('picture', { subType: 'carousel', number })` | `GET /api/picture/carousel?number=..` |

**带搜索关键词：**

```js
fetchItemList('video', { sort, page, element, q: '关键词' })
// → GET /api/video?sort=..&page=..&element=..&q=关键词
```

## 二、视频相关

### 2.1 轮播图

```
GET /api/picture/carousel?number=5
调用方式：fetchItemList('picture', { subType: 'carousel', number: 5 })
```

**CarouselItem：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `number` | 图片唯一标识 |
| `src` | `string` | 图片 URL |

### 2.2 视频列表（首页推荐）

```
GET /api/video/main?page=1&element=16
调用方式：fetchItemList('video', { subType: 'main', page: 1, element: 16 })
```

### 2.3 视频详情

```
GET /api/video/:vid
调用方式：fetchItem('video', vid)
```

### 2.4 相关视频推荐

```
GET /api/video/related?vid=123&sort=-date&page=1&element=5
调用方式：fetchItemList('video', { subType: 'related', vid, sort, page, element })
```

### 2.5 VideoItem 数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `vid` | `number` | 视频 ID，路由参数 |
| `coverUrl` | `string` | 封面图 URL |
| `videoUrl` | `string` | 视频文件 URL（详情页才有） |
| `title` | `string` | 视频标题 |
| `viewCount` | `number \| string` | 播放量 |
| `messageCount` | `number \| string` | 评论数 |
| `duration` | `string` | 时长，格式 `"12:34"` |
| `date` | `string` | 上传日期 |
| `uploader` | `UploaderInfo` | 上传者信息 |
| `tagList` | `TagItem[]` | 标签列表（详情页才有） |

**UploaderInfo：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `uid` | `number` | 用户 ID |
| `userName` | `string` | 用户名 |
| `profilePictureUrl` | `string` | 头像 URL |

**TagItem：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `tid` | `number` | 标签 ID |
| `tagName` | `string` | 标签名称 |
| `likeCount` | `number` | 点赞数 |
| `favouriteCount` | `number` | 收藏数 |
| `commentCount` | `number` | 评论数 |

## 七、认证相关（src/apis/account.js）

### 7.1 登录 / 注册

```
POST /api/login              请求体：{ username, password }  响应：{ code:200, data:{ token, uid } }
POST /api/register           请求体：{ username, password }  响应：{ code:200, data:{ token, uid } }
```

登录成功后 `Cookies.set('token', ...)` + `Cookies.set('uid', ...)`。注册成功后自动调 login 实现自动登录。

### 7.2 获取当前用户 Profile

```
GET /api/user/profile/
需要 needToken: true → 实际发送 POST body:{ authorization: token }
响应：{ code:200, data: UserItem } 或 { code:401 }
```

## 九、消息系统（src/apis/account.js）

### 9.1 发送私信

```
POST /api/user/message/send
需要 needToken: true
请求体：{ dialogueMid, text }    dialogueMid=null 表示新建对话
```

### 9.2 数据结构

**DialogueItem：** mid, opponent(UserInfo), sentences(MessageItem[]), lastMessage, lastDate

**MessageItem：** text, sender(UserInfo|null，null表示自己发的), date

**NotificationItem：** nid, type('like'|'comment'|'follow'|'favourite'|'at'), text, from(UserInfo), targetType, targetId, parentType, parentId, date

## 十、统一交互 API（src/apis/content.js）

### 10.1 interact 函数

```js
const interact = (mediaType, action, mediaId) => { ... }
// mediaType: 'video'|'comment'|'user'|'essay'|'post'|'tag'
// 需要 needToken: true
```

### 10.2 ACTION_MAP

| action | method | 路径 | 说明 |
|--------|--------|------|------|
| `like` / `unlike` | POST/DELETE | `/api/:type/:id/like` | 点赞 |
| `favourite` / `unfavourite` | POST/DELETE | `/api/:type/:id/favourite` | 收藏 |
| `reshare` / `unreshare` | POST/DELETE | `/api/:type/:id/reshare` | 转发 |
| `dislike` / `undislike` | POST/DELETE | `/api/:type/:id/dislike` | 点踩（评论专用） |
| `follow` / `unfollow` | POST/DELETE | `/api/:type/:id/follow` | 关注（用户专用） |

### 10.3 ID_KEY_MAP（InteractionBar 组件）

```js
const ID_KEY_MAP = { video:'vid', essay:'eid', post:'pid', tag:'tid', user:'uid', comment:'cid' }
const mediaId = item?.[ID_KEY_MAP[mediaType]]
```

## 十二、搜索

搜索不是独立接口，在各资源列表接口上加 `q` 参数：

```
GET /api/video?q=关键词&sort=-date&page=1&element=16
```

Mock 层过滤逻辑：匹配 `item.title` 或 `item.uploader.userName`，过滤在排序和分页之前执行。
