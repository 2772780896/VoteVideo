## 二、开发历程

### 2.1 Tailwind 重构

| antd | Tailwind |
|------|----------|
| antd Upload（拖拽组件） | `<label>` 虚线框 + `<input type="file" hidden>` |
| antd Modal | `fixed` 定位黑色遮罩 + 白色卡片 |
| antd Form/Input/Button | Tailwind div + input + button |
| UploadText 组件 | 内联 `<textarea>` |

UploadText.jsx 删除——两个父组件各自内联 textarea，不单独抽组件。

canvas 截图封面保留（`canvas.getContext('2d').drawImage()` + `toBlob()` 是浏览器原生 API，toBlob 异步不阻塞主线程）。

### 2.2 上传 API 与拦截器修复

```js
const uploadContent = (type, data) =>
  request.post(`/api/upload/${type}`, data, { needToken: true })
```

**拦截器 Bug**：旧拦截器把 `config.data` 替换为 `{ authorization: token }`，覆盖上传的 `{ title, cover, ... }`。修复：展开合并 `config.data = { ...(config.data || {}), authorization: token }`。

### 2.3 浏览器 API 详解

**`URL.createObjectURL(file)`**：把 File 对象转成临时 blob URL，纯本地预览不经过服务器。适合大文件（视频）。

**`<input type="file">`**：原生弹出系统文件夹。`e.target.files` 是 FileList（类数组），需 `Array.from()` 转真数组。

**`canvas.getContext('2d').drawImage()`**：三步截视频帧 → `toBlob(callback, 'image/jpeg', 0.9)` → Blob → 包装为 `new File([blob], 'cover.jpg')` 存 state（异步，不阻塞主线程）。

**`FileReader.readAsDataURL(file)`**：异步读图片为 base64。必须在 `onload` 回调里取 `reader.result`（JS 不允许同步阻塞读硬盘）。**改造后已不再使用 FileReader，改为直接保存 File 对象。**

**`FileReader` vs `createObjectURL`**：

| | createObjectURL | FileReader |
|---|---|---|
| 返回 | `blob:http://...` 临时链接 | `data:image/png;base64,...` 字符串 |
| 适合 | 大文件（视频） | 小文件（封面图/多图） |

**`e.stopPropagation()`**：弹窗遮罩点空白关闭，内层卡片 `stopPropagation` 阻止冒泡。

### 2.4 表单提交

视频：title + description + 视频 File + 封面 File → FormData → `uploadContent('video', formData)`
文章：title + description → `uploadContent('essay', { title, description })`（纯 JSON）
动态：text + images File[] → FormData → `uploadContent('post', formData)`（文字和图片至少有一个）

`submitting` state 控制按钮禁用态和文字。

### 2.5 UploadPost 多图上传

```js
// FileList → 数组 → 限制数量
const files = Array.from(e.target.files).slice(0, 9 - images.length)

// filter 按索引删除（无 id，用索引标识）
```

## 三、文件上传改造（base64/JSON → FormData 多部分表单）

> 原先上传流程完全基于 JSON + base64/blob URL：前端把视频生成为 blob URL、图片转为 base64 字符串，以 JSON body 发送。改造后切换为标准 FormData 多部分表单上传，视频/图片以二进制文件传输到服务端磁盘，数据库存储相对 URL 路径。

### 3.1 改造前后对比

| 维度 | 改造前（JSON + base64） | 改造后（FormData + multer） |
|------|------------------------|---------------------------|
| 前端数据格式 | 视频 blob URL 字符串 + 封面 base64 字符串 | 原始 File 对象 + FormData |
| 请求格式 | `Content-Type: application/json` | `Content-Type: multipart/form-data`（axios 自动设置） |
| 后端接收 | `req.body.videoUrl`（字符串）、`req.body.cover`（base64） | `req.files.video[0]`（File）、`req.files.cover[0]`（File） |
| 文件存储 | 无（URL/base64 直接存 DB） | multer 存盘到 `uploads/video/`、`uploads/cover/`、`uploads/post/` |
| DB 存储内容 | base64 字符串 / blob URL | 相对 URL 路径（如 `/uploads/video/xxx.mp4`） |
| 文件访问 | 不可访问（blob URL 仅本地有效） | `http://localhost:3000/uploads/video/xxx.mp4`（express.static 服务） |

### 3.2 前端改造要点

**UploadVideo.jsx：**
- state 拆分：`videoFile`（File）、`coverFile`（File/Blob）、`videoPreview`（blob URL）、`coverPreview`（blob URL）
- 视频选择：保存原始 `File` 对象，`URL.createObjectURL(file)` 仅用于 `<video>` 预览
- 封面截图：`canvas.toBlob(callback)` 替代 `canvas.toDataURL()` → 得到 Blob → 包装为 `new File([blob], 'cover.jpg')`
- 本地封面：直接保存 `File` 对象（不再 `FileReader.readAsDataURL`）
- 提交：`new FormData()` → `append('video', videoFile)` + `append('cover', coverFile)` + `append('title', title)` + `append('description', description)`

**UploadPost.jsx：**
- `images` 从 `string[]`（base64 数组）改为 `{ file: File, preview: string }[]`
- `handleImageAdd`：直接保存 `File` 对象 + `URL.createObjectURL` 预览 URL
- 提交：`formData.append('images', img.file)` 逐个 append（同名字段多值 = 数组，与后端 `multer.array('images', 9)` 匹配）

**content.js：**
- `uploadContent` 增加 `timeout: 120000`（2 分钟），覆盖默认 5000ms
- `data` 参数可以是普通对象（JSON）或 FormData（axios 自动识别并设置 Content-Type）

### 3.3 后端改造要点

**app.js：** `app.use('/uploads', express.static(path.join(__dirname, 'uploads')))` — 让上传文件可通过 HTTP 访问

**uploadRoutes.js：** 在 `needToken` 和 controller 之间插入 multer 中间件：
- `/video` → `videoUpload`（`multer.fields([{name:'video'}, {name:'cover'}])`）
- `/post` → `postUpload`（`multer.array('images', 9)`）

**uploadController.js：**
- 新增 `toUrl(file)` 辅助函数：把 multer file 对象转为 `/uploads/{子目录}/{文件名}` 相对 URL
- `uploadVideo`：从 `req.files.video[0]` / `req.files.cover[0]` 取文件 → `toUrl()` 转 URL
- `uploadPost`：从 `req.files`（数组）→ `.map(toUrl)` 转 URL 数组

**uploadMiddleware.js：** 启动时 `fs.mkdirSync(dir, { recursive: true })` 自动创建 `uploads/video`、`uploads/cover`、`uploads/post` 子目录

### 3.4 数据流（改造后）

```
前端 File 对象 → new FormData() → POST multipart/form-data
  → multer 中间件存盘到 uploads/xxx/ → req.files
  → controller 从 req.files 取文件对象 → toUrl() 转相对 URL
  → service 将 URL 写入 Prisma（videoUrl/coverUrl/pictureList）
  → 返回 { vid/pid } 给前端
  → 前端通过 /uploads/xxx 访问文件（express.static 提供服务）
```

## 四、架构决策

### 4.1 createObjectURL vs FileReader

视频用 createObjectURL（不转码，大文件友好）。

> **改造后更新：** 封面截图改用 `canvas.toBlob()`（异步，不阻塞主线程），本地封面和帖子图片都直接保存 File 对象。`createObjectURL` 仅用于预览展示，不再参与数据传输。FileReader 不再使用。

### 4.2 发布与展示分离

Upload 管发布，Dynamic 管展示，职责清晰。
