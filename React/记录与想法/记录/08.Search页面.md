## 二、开发历程

### 2.1 mock 加搜索过滤

5 个 getList 文件全部加 `q` 参数：

```js
const q = url.searchParams.get('q')
let list = [...allData]
if (q) {
  const kw = q.toLowerCase()
  list = list.filter(item =>
    item.title?.toLowerCase().includes(kw) ||
    item.uploader?.userName?.toLowerCase().includes(kw)
  )
}
// 过滤在 sortData（排序+分页）之前，否则跨页匹配的内容会丢失
```

**`?.` 可选链**：`a?.b?.c` — 链条上任一环节是 null/undefined 就返回 undefined，不抛异常。

### 2.3 闭包与 depsKey — DataList 适配器原理

**为什么需要箭头函数适配器**：

```
useData 内部：requestFunc(sort, page, pageSize, ...params)
fetchItemList 签名：(resourceType, { sort, page, element, q })
```

适配器桥接签名差异：

```js
request={(s, p, e) => fetchItemList(type, { sort: s, page: p, element: e, q: keyword })}
         ↑来自useData实参                   ↑闭包，来自组件作用域
```

**`params={[type, keyword]}` 的作用**：为 depsKey 服务——keyword 变了 → depsKey 变了 → useEffect 触发 → 重新请求。闭包让函数知道值是多少，depsKey 让 useEffect 知道该不该调。

**设计反思**：当前 useData 把"传参"和"依赖"混在同一个 params 里。理想设计：`useData(requestFunc, deps, ...args)` 分开。

### 2.4 Bug 1：tab 切换崩溃

**正常流程**（同 tab 换关键词）：旧数据格式与新 Card 兼容 → 正常渲染旧数据等新数据替换 ✅

**异常流程**（切 tab，视频→动态）：

```
当前：data = [视频对象], Card = VideoCard

点"动态" → URL 变 → SearchPage 重跑
  Card = PostCard  ← 已变
  useData: data = [视频对象]  ← useState 返回旧值！（当前渲染周期不能修改 state）
  depsKey 变 → useEffect 排队，但还没执行

渲染：<PostCard post={视频对象} /> → 崩溃！
```

**React 三阶段原理**：

```
① Render Phase（同步）→ 跑函数体 → 读 state → 生成 Element → 崩了 → 中止
② Commit Phase（同步）→ DOM 操作 → 第①步崩了不会执行
③ Effect Phase（异步）→ useEffect → 第①步崩了不会执行
```

**崩溃于第①步末尾**。useEffect 里 setLoading(true) 没有机会触发，骨架屏永远不会出现。

**为什么 key 能救**：`<DataList key="posts" />` — key 变了，React 认为是新组件 → 旧实例卸载 + 新实例挂载 → useState(null) 真正返回 null → 渲染骨架屏 ✅

**为什么 Vue 不会**：Vue 的响应式系统 + 同步 watch 可在渲染前清空 data。React 的函数式 + 异步 effect 在"props 变导致 state 形状不兼容"时需要额外防御（key）。

### 2.5 Bug 2：竞态条件

```
搜 "严"   → 发请求 A（慢，3秒）
搜 "严丰" → 发请求 B（快，0.5秒）
B 先返回 → setData ✅
A 后返回 → setData 覆盖 B ❌
```

**修复：requestId + useRef**

```js
const requestIdRef = useRef(0)
async function getData() {
    const currentId = ++requestIdRef.current
    setLoading(true)
    const response = await requestFunc(...params)
    if (currentId !== requestIdRef.current) return  // 过期，丢弃
    setData(response.data)
}
```

**useRef 原理**：`useRef(0)` 返回 `{ current: 0 }`，存在于 React 状态系统外部。改它不触发重渲染，重渲染也不重置。就像在组件外面挂了个小盒子。

> useRef 的详细说明见 [通用组件与Hook.md](./10.通用组件与Hook.md) 第四节。

## 三、架构决策

### 3.1 key 强制 remount vs useEffect 检测

tab 切换时 Card 类型变了，旧 state 形状不兼容。用 `key={activeKey}` 让 React 卸载重装，比在 useEffect 里手动清空更简洁。

### 3.2 闭包 + params 协作

闭包让函数知道值，params/depsKey 让 useEffect 知道该不该调。两者缺一不可。

## 六、核心认知

**useState 只在首次挂载时用初始值**。之后每次重渲染，React 从内部存储取上一次 set 的值。初值参数仅用来初始化，不是"每次都重置"。
