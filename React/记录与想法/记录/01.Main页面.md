## 二、开发历程

### 2.1 接口定义与数据流搭建

首先定义了两个接口的请求/响应格式，然后从 Mock → API 层 → axios → useData → 页面组件，逐层打通数据流：

```
① Mock.js 拦截 → ② apis/content.js → ③ request.js → ④ useData → ⑤ Main.jsx → ⑥ 渲染
```

每一步的数据形态：

| 步骤 | 文件 | 数据形态 |
|------|------|---------|
| ① Mock | `mock/response/carouselPicture.js` | `{ code:200, data:[{id, src}, ...] }` |
| ② API | `apis/content.js` | 返回 axios Promise |
| ③ axios | `utils/request.js` | 拦截后的 `response` 对象 |
| ④ useData | `hooks/useData.js` | `{ data: response.data, loading, error }` |
| ⑤ Main | `pages/Main.jsx` | `carouselUrlList`, `mainVideoList` |

useData 返回值到组件的映射：

```
useData(...)
  → { data: response.data, loading, error }
    → carouselData = { code:200, data:[{id, src}] }
      → carouselUrlList = carouselData?.data = [{id, src}]
        → <Carousel urlList={carouselUrlList} />
```

### 2.2 useData Hook 设计

**为什么需要 useData**：直接用 `useEffect + useState` 每个页面都要写一遍重复的状态逻辑。useData 封装成黑盒，调用方只需 `const { data, loading, error } = useData(fn, ...params)`。

**旧版的问题**：
- 初始值 `{ data: [] }` 和请求成功 `{ data: [...] }` 形式一样，无法区分"未请求"和"空数据"
- catch 里 `setData(error.data)` 为 undefined，`.map()` 崩溃
- 没有 loading 状态，按钮可重复点击

**新版 v2**：data 初始为 `null`，error 独立存储，loading 完整覆盖。

> useData 的完整设计、后续演进（refresh、depsKey、竞态修复）见 [通用组件与Hook.md](./通用组件与Hook.md) 第四节。

### 2.3 Carousel 组件实现

内联在 Main.jsx 中，核心逻辑：

```jsx
function Carousel({ urlList }) {
  const [current, setCurrent] = useState(0)

  // 自动轮播：urlList 变化时重新设 timer
  useEffect(() => {
    if (!urlList || urlList.length === 0) return
    const timer = setInterval(() => {
      setCurrent(prev => (prev + 1) % urlList.length)
    }, 3000)
    return () => clearInterval(timer)  // 防内存泄漏
  }, [urlList])

  // 切换：+1/-1 取模，处理负数边界
  const prev = () => setCurrent(prev => (prev - 1 + urlList.length) % urlList.length)
  const next = () => setCurrent(prev => (prev + 1) % urlList.length)
  // ... 图片用 translateX 移动 + CSS transition 动画
}
```

### 2.5 五状态 UI 覆盖

条件链顺序必须严格：`loading → error → 空态 → 正常渲染`。

```jsx
{loading ? 骨架屏 : error ? 错误提示 : list?.length === 0 ? 空状态 : 渲染}
```

**短路求值是顺序不能乱的根本原因**：loading=true 时 `error?` 不会执行。如果把 `length===0` 放最前面，loading 期间 list 是 `null`，`null===0` 为 false，会错误渲染导致崩溃。

### 2.6 响应式 Grid 布局

```jsx
<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-5 justify-items-center">
```

| 前缀 | 触发条件 | 列数 | 设备 |
|------|----------|------|------|
| 无 | `< 640px` | 1 | 手机 |
| `sm:` | `≥ 640px` | 2 | 大手机 |
| `lg:` | `≥ 1024px` | 3 | 平板 |
| `xl:` | `≥ 1280px` | 4 | 桌面 |

**Grid vs Flex**：Flex 是一维排列（导航栏、卡片内部），Grid 是二维网格（视频列表）。`justify-items-center` 让最后一行不满时居中对齐。

### 2.7 骨架屏

```jsx
Array(16).fill(0).map((_, i) => (
  <div key={i} className="w-[300px] animate-pulse">
    <div className="aspect-video rounded-xl bg-gray-200 mb-2" />
    <div className="h-4 bg-gray-200 rounded w-3/4 mb-1" />
    <div className="h-4 bg-gray-200 rounded w-1/2" />
  </div>
))
```

- `Array(16).fill(0)` — 稀疏数组的 map 不执行回调，fill 转为密集数组
- 数量 16 与真实数据量一致，保持 CLS=0（Cumulative Layout Shift）
- `animate-pulse` 底层是 opacity 的 ease-in-out 关键帧动画

## 三、架构决策

### 3.1 useData 初始 data 用 null 而非 []

`null` = "还没请求" → 骨架屏；`[]` = "请求成功但数据为空" → 空状态。两者语义不同。

### 3.2 `setLoading(true)` 和 `setError(null)` 写在 try 前面

React 18 Automatic Batching 合并同一执行上下文的 setState。如果先设 error 再设 loading，中间会短暂出现 "loading=false + error=旧值" 的一帧，UI 闪烁。
