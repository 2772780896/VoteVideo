## 一、页面职责

动态详情页，展示用户发布的动态内容（文本/图片/视频的任意组合），支持点赞/收藏/转发和评论区。

接口：`fetchItem('post', pid)` 获取详情。字段定义见 [前端接口分析.md](./13.%20VoteVideo-前端接口分析.md) 第二节。

## 二、开发历程

### 2.1 从 type 字段到 slots 模式

**旧设计**：`post.type` 为单值字符串（'video'|'text'|'picture'），if/else 三分支渲染。80% 的代码在每个分支中重复。

**新设计**：三种内容位置平行预分配，根据数据有无决定显隐：

```jsx
{/* 文本区 — 有就亮 */}
{post.text && <p>{post.text}</p>}

{/* 图片区 — 有就亮，网格排列 */}
{post.pictureList?.length > 0 && (
  <div className="grid grid-cols-2 md:grid-cols-3 gap-3">
    {post.pictureList.map((src, i) => (
      <img key={i} src={src} className="w-full aspect-square object-cover rounded-lg" />
    ))}
  </div>
)}

{/* 视频区 — 有就亮 */}
{post.videoList?.length > 0 && <VideoCard video={post.videoList[0]} />}

{/* 固定区 */}
<InteractionBar mediaType="post" item={post} />
```

**优点**：天然支持复合类型（文本+图片+视频共存），零 if/else，废弃 PostVideo/PostPicture 子组件。

### 2.2 Mock 数据改造

```js
// 旧：roll 单一 type
type: Random.pick(['video', 'text', 'picture'])

// 新：三块内容独立决定有无
text:        Random.boolean() ? Random.cword(50, 200) : null,
videoList:   Random.boolean() ? createVideo(1) : null,
pictureList: Random.boolean() ? [...Array(Random.integer(1, 6))].map(...) : null,
```

### 2.3 其他改造

- `useData(fetchItem, 'post', pid)` 四状态解构
- antd → Tailwind
- `<InteractionBar mediaType="post" item={post} />`

## 三、架构决策

### 3.1 slots 模式 vs type 分支

| | type 分支 | slots 模式 |
|---|---|---|
| 复合类型 | 不支持（只能选一种） | 天然支持 |
| 代码重复 | 80% 重复 | 零重复 |
| 新增内容类型 | 加分支 + 改条件 | 加一个 `{xxx && ...}` |
| 组件废弃 | PostVideo/PostPicture | 不再需要 |
