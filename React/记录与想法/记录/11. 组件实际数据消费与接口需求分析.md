## 一、数据流向分析

### 1.1 总体架构

```
┌─────────────────────────────────────────────────────────────┐
│                       前端架构                              │
├─────────────────────────────────────────────────────────────┤
│  Pages (页面层)                                          │
│    ↓ 调用                                                 │
│  APIs (api层: content.js, account.js)                   │
│    ↓ axios请求                                            │
│  Utils/request.js (请求封装、响应拦截器、错误处理)         │
│    ↓                                                     │
│  后端API                                                 │
├─────────────────────────────────────────────────────────────┤
│  组件间数据传递：                                         │
│  - useData Hook: 管理请求状态 (data/loading/error/refresh)│
│  - DataList: 封装分页逻辑，渲染列表                      │
│  - 直接props传递: 父子组件通信                           │
└─────────────────────────────────────────────────────────────┘
```

## 二、组件数据消费格式分析

### 2.1 Video 数据格式 (组件实际消费)

**消费位置**: `Video.jsx`, `VideoCard.jsx`, `VideoPlayerApp.jsx`

```javascript
{
  vid: number,              // 视频 ID (必填)
  title: string,            // 标题 (必填)
  coverUrl: string,         // 封面 URL (必填)
  videoUrl: string,         // 视频文件 URL (详情页必填)
  viewCount: number,        // 播放量 (必填)
  commentCount: number,      // 评论数 (已统一，原messageCount)
  duration: string,         // 时长 "mm:ss" (必填)
  date: string,             // 日期 "yyyy-MM-dd"
  likeCount: number,
  favouriteCount: number,
  uploader: {              // 上传者信息 (必填)
    uid: number,
    userName: string,
    profilePictureUrl: string
  },
  tagList: [{              // 标签列表 (详情页)
    tid: number,
    tagName: string
  }]
}
```

### 2.2 Essay 数据格式

**消费位置**: `Essay.jsx`, `EssayCard.jsx`

```javascript
{
  eid: number,              // 文章 ID (必填)
  title: string,            // 标题 (必填)
  coverUrl: string,         // 封面 URL
  text: string,             // 正文 (必填)
  viewCount: number,
  commentCount: number,     // 评论数
  uploader: {              // 上传者信息 (必填)
    uid: number,
    userName: string,
    profilePictureUrl: string
  },
  date: string,
  likeCount: number,
  favouriteCount: number
}
```

### 2.3 Post 数据格式 (复合类型)

**消费位置**: `Post.jsx`, `PostCard.jsx`

```javascript
{
  pid: number,              // 动态 ID (必填)
  title: string | null,    // 标题 (可选)
  text: string | null,     // 文本内容 (可选)
  pictureList: string[] | null,  // 图片列表 (可选)
  videoList: [{             // 视频列表 (可选)
    vid: number,
    title: string,
    coverUrl: string,
    duration: string,
    viewCount: number,
    commentCount: number,
    uploader: {...}
  }] | null,
  viewCount: number,
  commentCount: number,
  uploader: {              // 上传者信息 (必填)
    uid: number,
    userName: string,
    profilePictureUrl: string
  },
  date: string,
  likeCount: number,
  favouriteCount: number
}
```

**注意**: Post 是复合类型，可包含文本、图片、视频中的任意组合。

### 2.4 Comment 数据格式 (递归结构)

**消费位置**: `CommentCard.jsx`, `Video.jsx`, `Essay.jsx`, `Tag.jsx`

```javascript
{
  cid: number,              // 评论 ID (必填)
  vid: number,              // 所属视频/文章/动态 ID
  uploader: {              // 评论者信息 (必填)
    uid: number,
    userName: string,
    profilePictureUrl: string
  },
  replyTo: {               // 回复目标 (可选)
    uid: number,
    userName: string,
    profilePictureUrl: string
  } | null,
  text: string,             // 评论内容 (必填)
  type: 'text' | 'picture', // 评论类型
  pictureList: string[],    // 图片列表 (type=picture时)
  viewCount: number,
  likeCount: number,
  favouriteCount: number,
  subCommentCount: number,  // 子评论数
  subCommentList: [Comment], // 子评论 (递归，可选)
  date: string              // 日期 (必填)
}
```

## 四、API层与实际需求的匹配分析

### 4.1 已定义的API (content.js)

```javascript
// 获取单个资源
fetchItem(resourceType, id) 
  → GET /api/{resourceType}/{id}

// 获取资源列表
fetchItemList(resourceType, params)
  → GET /api/{resourceType} 或 /api/{resourceType}/{subType}

// 上传内容
uploadContent(type, data)
  → POST /api/upload/{type}

// 统一交互
interact(mediaType, action, mediaId)
  → POST/DELETE /api/{mediaType}/{mediaId}/{action}
```

**SUBTYPE_MAP** (子资源URL映射):
- `video/main` → `/api/video/main`
- `video/related` → `/api/video/related`
- `picture/carousel` → `/api/picture/carousel`

**ACTION_MAP** (交互动作映射):
- `like/unlike` → `/api/{mediaType}/{mediaId}/like`
- `favourite/unfavourite` → `/api/{mediaType}/{mediaId}/favourite`
- `reshare/unreshare` → `/api/{mediaType}/{mediaId}/reshare`
- `dislike/undislike` → `/api/{mediaType}/{mediaId}/dislike`
- `follow/unfollow` → `/api/{mediaType}/{mediaId}/follow`

### 4.2 已定义的API (account.js)

```javascript
// 登录/注册
login(username, password) → POST /api/login
register(username, password) → POST /api/register

// 用户资料
getMyProfile() → GET /api/user/profile/
getUserPublicProfile(uid) → GET /api/user/{uid}

// 个人中心子数据
getProfileSubdata(sort, page, element, profileType, dataType)
  → GET /api/user/profile/{profileType}/{dataType}?sort,page,element

// 发送私信
sendMessage(dialogueMid, text) → POST /api/user/message/send
```

## 五、应该实现的接口清单 (后端)

根据前端组件的实际需求，后端应该实现以下接口：

### 5.1 内容获取接口

#### 视频相关
- `GET /api/video/:vid` - 获取视频详情
- `GET /api/video` - 获取视频列表 (支持query参数: page, element, sort, uid, q)
- `GET /api/video/main` - 获取首页推荐视频
- `GET /api/video/related` - 获取相关视频

#### 文章相关
- `GET /api/essay/:eid` - 获取文章详情
- `GET /api/essay` - 获取文章列表 (支持query参数: page, element, sort, uid, q)

#### 动态相关
- `GET /api/post/:pid` - 获取动态详情
- `GET /api/post` - 获取动态列表 (支持query参数: page, element, sort, uid, q)

#### 评论相关
- `GET /api/comment` - 获取评论列表 (支持query参数: vid, eid, pid, sort, page, element)
- `POST /api/comment` - 发表评论
- `DELETE /api/comment/:cid` - 删除评论

#### 标签相关
- `GET /api/tag/:tid` - 获取标签详情
- `GET /api/tag` - 获取标签列表 (支持搜索)
- `GET /api/tag/hot` - 获取热门标签 (建议实现)

#### 用户相关
- `GET /api/user/:uid` - 获取用户公开资料
- `GET /api/user/:uid/video` - 获取用户上传的视频
- `GET /api/user/:uid/essay` - 获取用户上传的文章
- `GET /api/user/:uid/post` - 获取用户发布的动态

### 5.2 内容上传接口

- `POST /api/upload/video` - 上传视频
- `POST /api/upload/essay` - 上传文章
- `POST /api/upload/post` - 发布动态
- `PUT /api/upload/video/:vid` - 编辑视频 (建议实现)
- `PUT /api/upload/essay/:eid` - 编辑文章 (建议实现)
- `PUT /api/upload/post/:pid` - 编辑动态 (建议实现)
- `DELETE /api/upload/video/:vid` - 删除视频 (建议实现)
- `DELETE /api/upload/essay/:eid` - 删除文章 (建议实现)
- `DELETE /api/upload/post/:pid` - 删除动态 (建议实现)

- `DELETE /api/post/:pid/reshare` - 取消转发
